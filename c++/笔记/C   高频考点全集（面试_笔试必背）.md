# 一、C++ 基础核心（必考题）

## 1. 变量与数据类型

- 基本类型：int, char, float, double, bool, long, long long
- 类型大小（64位系统）：char=1字节，int=4字节，float=4字节，double=8字节，指针=8字节
- 常量：const（编译期/运行期常量，有类型检查）、#define（宏替换，无类型检查，预处理阶段生效）
- typedef vs using：两者均可定义类型别名，using更现代，支持模板（如template<typename T> using MyVec = vector<T>），typedef不支持模板

## 2. 引用与指针（高频）

### （1）指针

- 本质：存放内存地址的变量，可指向任意合法内存，也可指向空（nullptr）
- 指针运算：p++/p-- 移动一个对应类型的内存大小（如int* p，p++移动4字节）
- 关键问题：野指针（指向已释放、无效内存，需避免，如未初始化的指针、释放后未置空的指针）
- 空指针：C++11新增nullptr，替代NULL（NULL本质是0，存在类型歧义，nullptr是指针类型，更安全）

### （2）引用

- 本质：变量的别名，底层通过指针实现，使用时自动解引用，无需手动操作
- 核心要求：必须初始化（绑定一个已存在的变量），不能为空，不能重新绑定其他变量
- 常引用：const int&，可绑定临时变量（如const int& a = 10; 合法），普通引用不能绑定临时变量

### （3）核心区别（必背）

1. 初始化：指针可空、可后期赋值；引用必须初始化，不能为空
2. 指向性：指针可修改指向的内存地址，支持多级指针（如int** p）；引用不可重新绑定，无多级引用
3. 使用：指针需用*解引用、->访问成员；引用直接使用，无需解引用

## 3. 函数基础

- 函数重载：同一作用域内，函数名相同，参数的个数、类型、顺序不同（与参数名、返回值无关），编译器根据参数匹配调用
- 缺省参数：参数可设置默认值，必须从右往左定义（如void func(int a, int b=2, int c=3); 合法；void func(int a=1, int b); 非法），调用时从左往右传参，缺省的参数可省略
- 内联函数 inline：        
  - 作用：编译时将函数体展开到调用处，减少函数调用的开销（如压栈、跳转）
  - 适用场景：短小、频繁调用的函数（如简单的加法、判断），过长的函数会被编译器优化为普通函数
  - 与宏的区别：有类型检查、安全可调试，宏无类型检查，仅文本替换，易出错

## 4. 三大特性（面向对象核心，基础铺垫）

1. 封装：隐藏类的实现细节，只暴露公共接口，通过private（仅类内访问）、protected（类内+子类访问）、public（类内+类外访问）控制访问权限，提高代码安全性和可维护性
2. 继承：实现代码复用，子类（派生类）继承父类（基类）的成员和方法，语法：class 派生类名 : 继承方式 基类名（如class B : public A）
3. 多态：基类指针/引用指向派生类对象，调用同名方法时，根据对象的实际类型执行不同的实现，是面向对象的核心特性

# 二、C++ 核心进阶（面试重灾区）

## 1. 类与对象

- 类的6个默认成员函数（必考）：编译器自动生成，当用户手动实现任意一个时，编译器不再生成其他默认函数        
  - 构造函数：初始化对象，无返回值，可重载，函数名与类名相同（如class A { A() {}; }）
  - 析构函数：释放对象资源，无参无返回值，只能有一个，函数名前加~（如~A() {}）
  - 拷贝构造函数：用一个已存在的对象初始化新对象，原型：A(const A& other)
  - 赋值运算符重载：将一个对象的值赋给另一个对象，原型：A& operator=(const A& other)
  - 取地址运算符重载：返回对象的地址，原型：A* operator&()
  - const取地址运算符重载：返回const对象的地址，原型：const A* operator&() const
- 访问权限：牢记三个权限的访问范围，面试常考区分

## 2. 构造/析构/拷贝（必考）

### （1）构造函数

- 核心：用于初始化对象，可重载（如无参构造、有参构造、拷贝构造）
- 初始化列表：必须使用的场景——const成员变量、引用成员变量、无默认构造的基类/成员对象
  - 注意：初始化顺序与成员变量在类中的声明顺序一致，与初始化列表的顺序无关
  - 语法：类名(参数) : 成员1(值1), 成员2(值2) { 函数体 }

### （2）析构函数

- 核心：释放对象占用的资源（如new分配的内存），程序结束时自动调用
- 关键：多态场景下，基类的析构函数必须设为虚函数，否则会导致内存泄漏

### （3）拷贝构造

- 浅拷贝：编译器默认生成的拷贝构造，仅拷贝成员变量的值，若成员有指针，会导致两个对象指向同一块内存，析构时双重释放
- 深拷贝：用户手动实现，给指针成员重新开辟内存，拷贝指针指向的内容，避免双重释放，适用于类中有动态内存成员的场景

### （4）赋值运算符重载

- 核心步骤：先判断自赋值（if(this == &other) return *this;），再释放自身旧资源，然后拷贝对方资源，最后返回*this（支持连续赋值，如a = b = c）

## 3. 继承（高频）

- 继承方式：public（基类public→派生类public，protected→protected，private→private）、protected（基类public/protected→派生类protected）、private（基类所有权限→派生类private）
- 赋值兼容规则：派生类对象可以赋值给基类对象、基类指针、基类引用（向上转型），反之不成立
- 菱形继承：多个派生类继承同一个基类，又被同一个子类继承，导致二义性和数据冗余，解决方案：虚继承（class B : virtual public A），通过虚基类指针解决
- 构造/析构顺序：构造→基类→成员对象→派生类；析构→派生类→成员对象→基类（与构造顺序相反）

## 4. 多态（最高频考点）

### （1）多态成立条件（必背）

1. 基类定义虚函数（用virtual修饰）
2. 派生类重写虚函数（函数名、参数、返回值完全相同，派生类可省略virtual）
3. 基类指针/引用指向派生类对象

### （2）重写/重载/重定义（必区分）

- 重载：同一作用域，同名不同参（个数/类型/顺序），与返回值无关
- 重写（覆盖）：不同作用域（基类+派生类），同名同参同返回值，基类带virtual
- 重定义（隐藏）：派生类与基类同名函数（无论参数是否相同），屏蔽基类同名函数

### （3）虚函数底层

- 含虚函数的类，编译器会生成一张虚函数表（_vftable），存储类中所有虚函数的地址
- 类的每个对象会占用8字节（64位系统）存储虚表指针（_vptr），指向虚函数表
- 多态的本质：运行时通过虚表指针找到对应对象的虚函数，实现动态绑定

### （4）纯虚函数

- 语法：virtual void func() = 0;
- 包含纯虚函数的类是抽象类，不能实例化，必须由派生类重写纯虚函数后，派生类才能实例化

## 5. 内存管理（高频）

### （1）new/delete vs malloc/free（必背）

1. 本质：new/delete是C++运算符；malloc/free是C语言库函数，需包含<cstdlib>
2. 内存计算：new自动计算类型大小；malloc需手动指定（如malloc(sizeof(int))）
3. 构造/析构：new调用构造函数，delete调用析构函数；malloc/free仅分配/释放内存，不调用
4. 返回值：new返回对应类型指针；malloc返回void*，需强制转换
5. 错误处理：new失败抛bad_alloc异常；malloc失败返回NULL

### （2）内存分布（必背）

1. 栈：局部变量、函数参数、返回值，自动分配释放，空间小（几MB），速度快
2. 堆：new/malloc分配的内存，手动释放，空间大（几十GB），速度慢
3. 数据段：全局变量、静态变量（static），程序启动分配，结束释放
4. 代码段：函数体、常量，只读，程序启动加载

# 三、C++11 新特性（必考）

- nullptr：替代NULL，类型安全，避免NULL的歧义（NULL本质是0）
- auto/decltype：auto自动推导变量类型（如auto a = 10; 推导为int）；decltype推导表达式类型（如decltype(a+b) c;）
- 范围for：遍历容器/数组，语法：for(auto x : 容器/数组) { ... }
- 右值引用：用&&表示，指向临时对象（右值），实现移动语义、完美转发，提高效率
- 移动构造/移动赋值：转移临时对象的资源，不进行拷贝，减少内存开销
- lambda表达式：匿名函数，语法：[捕获列表](参数列表)->返回值类型 { 函数体 }，适用于短期使用的简单函数
- 智能指针：unique_ptr（独占所有权）、shared_ptr（共享所有权，引用计数）、weak_ptr（解决循环引用）
- constexpr：编译期常量，在编译期确定值，提高效率
- 其他：列表初始化（如vector<int> v = {1,2,3};）、委托构造、继承构造

# 四、STL 高频考点

## 1. 常用容器底层（必背）

| 容器          | 底层结构                 | 访问效率           | 插入删除效率       |
| :------------ | :----------------------- | :----------------- | :----------------- |
| vector        | 动态数组（连续内存）     | 随机访问O(1)       | 尾插O(1)，中间O(n) |
| list          | 双向链表（不连续内存）   | 顺序访问O(n)       | 任意位置O(1)       |
| deque         | 双端队列（分段连续内存） | 随机访问O(1)       | 头尾插入删除O(1)   |
| string        | 字符数组（连续内存）     | 随机访问O(1)       | 尾插O(1)，中间O(n) |
| map/multimap  | 红黑树（平衡二叉搜索树） | O(logn)            | O(logn)            |
| set/multiset  | 红黑树                   | O(logn)            | O(logn)            |
| unordered_map | 哈希表                   | 平均O(1)，最坏O(n) | 平均O(1)，最坏O(n) |
| unordered_set | 哈希表                   | 平均O(1)，最坏O(n) | 平均O(1)，最坏O(n) |

## 2. 迭代器

- 迭代器失效：vector、deque插入/删除元素后，迭代器可能失效（指向已释放内存），需重新获取迭代器
- 迭代器分类：输入迭代器（只读）、输出迭代器（只写）、前向迭代器（可读写，单向）、双向迭代器（可读写，双向，如list）、随机访问迭代器（可读写，随机访问，如vector）

## 3. 常用算法

- sort：默认升序，底层是快速排序（数据量大）+插入排序（数据量小），头文件<algorithm>
- find：顺序查找，返回第一个匹配元素的迭代器，找不到返回end()
- 注意：STL算法不修改容器大小，仅操作容器内元素

# 五、高级考点（大厂必考）

## 1. 智能指针（必背）

- unique_ptr：独占所有权，不能拷贝，只能通过std::move转移所有权，效率最高，无引用计数
- shared_ptr：共享所有权，通过引用计数管理内存，每拷贝一次计数+1，析构一次计数-1，计数为0释放内存
- weak_ptr：不拥有所有权，不增加引用计数，用于解决shared_ptr的循环引用问题（如A和B互相持有shared_ptr），可通过lock()获取shared_ptr访问内存

## 2. 四大类型转换（必背）

- static_cast：最常用，用于基础类型转换（int→double）、上行转型（派生类→基类），无安全检查，编译期完成
- dynamic_cast：仅用于多态类型向下转型（基类→派生类），有安全检查（失败返回NULL），运行期完成，需基类有虚函数
- const_cast：仅用于去掉变量的const属性（如const int& → int&），不能修改非const变量的const属性
- reinterpret_cast：强制二进制转换，最危险（如int*→char*），仅用于底层操作，不推荐使用

## 3. 模板

- 函数模板：template<typename T> T func(T a, T b) { return a + b; }
- 类模板：template<typename T> class MyClass { T data; };
- 模板特化：全特化（指定所有模板参数）、偏特化（指定部分模板参数），用于特殊类型的定制实现
- 编译特性：二次编译，模板的声明和定义必须放在同一文件（如.h文件），否则会编译报错

## 4. 单例模式（必考手写）

- 饿汉式：程序启动时创建对象，线程安全，优点简单，缺点浪费内存        `class Singleton { ``private: ``    static Singleton instance; // 程序启动时初始化 ``    Singleton() {} // 私有构造，禁止外部实例化 ``public: ``    static Singleton& getInstance() { ``        return instance; ``    } ``};`
- 懒汉式（最优写法，C++11）：第一次使用时创建，静态局部变量初始化线程安全，节省内存        `class Singleton { ``private: ``    Singleton() {} ``public: ``    static Singleton& getInstance() { ``        static Singleton instance; // 仅初始化一次 ``        return instance; ``    } ``};`

## 5. 线程与并发

- 核心组件：thread（线程）、mutex（互斥锁，防止线程竞争）、condition_variable（条件变量，线程同步）、atomic（原子变量，无锁同步）
- 死锁条件（必背）：互斥、请求保持、不可剥夺、循环等待
- 死锁解决：按序加锁、超时锁、一次性申请所有资源
