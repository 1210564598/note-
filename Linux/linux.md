# Linux基础

## 一、名词解释

### 1. 家目录

- **定义**：用户登录后默认的工作目录

- 完整路径：

  ```
  /home/用户名
  ```

- **简写符号**：`~`（只代表家目录）

- **特点**：每个用户独立，权限私有

### 2. cd 命令（切换目录）

- `cd` / `cd ~`：回到当前用户**家目录**
- `cd -`：回到**上一次所在目录**
- `pwd`：显示当前完整路径（print working directory）

### 3. 终端提示符格式

- 用户名@主机名:当前路径$

例：`itcast@itcast:~$`

- `$`：普通用户
- `#`：root 超级管理员

### 4. 根目录常用子目录

- `/bin`：存放可执行命令（date、ls、cat、echo）
- `/dev`：硬件设备文件（一切皆文件）
- `/etc`：系统 / 软件配置文件
- `/home`：存放所有普通用户的家目录

### 5.常用命令解析器

1. **sh**：/bin/sh（基础）
2. **bash**：/bin/bash（增强，常用）
3. **查看当前 shell**：echo $SHELL
4. **查看所有 shell**：cat /etc/shells
5. **命令存放目录**：/bin
6. ctrl +l  清屏

------

## 二、root 用户（超级管理员）

- **切换命令**：`sudo su`（需输密码）
- **切换后**：`$` 变为 `#`
- **root 家目录**：`/root`（不是 /home）
- **权限**：最高权限，可操作所有文件
- **注意**：慎用，误删易导致系统崩溃

------

## 三、退出 root 用户

- **退出命令**：`exit`
- 退出后：`#` 变回 `$`，回到普通用户

------

## 四、 Tab 键功能

命令补齐：输入前缀按 Tab 自动补全（his→history）

文件补齐：命令后按 Tab 显示当前目录匹配文件

作用：减少输入、提升速度、避免输错	

## 五、常用快捷键

历史命令：上翻 **Ctrl+P**、下翻 **Ctrl+N**

光标移动：左 **Ctrl+B**、右 **Ctrl+F**、行首 **Ctrl+A**、行尾 **Ctrl+E**

删除操作：删前字符 **Ctrl+H**、删后字符 **Ctrl+D**、删光标前 **Ctrl+U**、删光标后 **Ctrl+K**

示例

```
Tab                   //自动补齐命令、文件名、路径（1次补全，2次列出）
Ctrl + P              //上一条历史命令（同向上箭头）
Ctrl + N              //下一条历史命令（同向下箭头）
Ctrl + A              //光标快速移动到行首
Ctrl + E              //光标快速移动到行尾
Ctrl + D              //删除光标所在位置字符（等于Delete键）
Ctrl + U              //清空光标到行首的所有内容
```

# Linux 根目录下的的子目录

### 1. /bin**

- **作用**：存放**常用二进制可执行命令**

- **例子**：`date`、`ls`、`cat`、`echo`

  示例 + 结果：bash

  ```
  cd /bin       # 进入bin目录
  ls            # 查看内容
  # 结果：date ls cat echo 等文件
  ./date        # 直接执行date
  # 结果：显示当前系统时间
  ```

### 2. /boot

- **作用**：存放**系统启动文件**
- **内容**：内核、引导程序（grub）
- **注意**：**不能乱删**，否则系统无法启动

### 3. /dev**

- ##### **<u>核心思想：一切皆文件</u>**

- **作用**：硬件设备对应**虚拟文件**

- 例子：

  - `/dev/sda`：第一块硬盘
  - `/dev/mem`：内存
  - `/dev/input/mice`：**鼠标**

- 测试命令：bash运行

  ```
  sudo cat /dev/input/mice
  ```

  输入密码   移动鼠标会显示数据。

### 4. /**etc ****

1. 作用系统配置文件目录，所有系统 / 软件的**<u>配置</u>**都在这里。

2. 典型路径

   - 网络配置：`/etc/network`
   - MySQL 配置：`/etc/mysql`（后缀 `.cnf`）
   - Nginx、Redis 等服务配置

3. 示例命令 + 结果

   ```
   ls /etc  //结果：显示 mysql、network、nginx 等配置文件夹。
   ```

### 5. /home 目录（宿主目录 / 家目录）**

1. 作用存放每个用户的专属目录，一人一个独立文件夹。

2. 结构

   - 格式：`/home/用户名`
   - itcast 用户 → `/home/itcast`
   - lisi 用户 → `/home/lisi`

3. 重点

   - **家目录 = /home/ 用户名**，不是 `/home` 本身
   - 用户登录后**默认进入自己的家目录**

4. 示例命令 + 结果

   ```
   ls /home //结果：（显示系统内所有用户目录）
   ```

### 6. /lib **

1.**全称**：library（系统库文件目录）

2.*作用**：存放 Linux 系统运行**必须的动态库文件**

3.*位置**：根目录下 `/lib`

4.重要子目录（重点）

`/lib/x86_64-linux-gnu`

- **说明**：64 位 Ubuntu 系统**核心库目录**
- **存放**：标准 C 库、线程库、命令行编辑库等系统依赖库

5.常见库文件（必背）

- `libc.so.2.27` → **标准 C 库**（系统最基础库）
- `libpthread.so.0` → 线程库
- `libreadline.so.7` → 命令行编辑库

6.、命令示例 + 执行结果

```
cd /lib/x86_64-linux-gnu //进入 lib 核心库目录
```

```
ls libc.so* //查看标准 C 库文件  //结果libc.so.2.27
```

```
ls libpthread.so* //查看线程库 //结果libpthread.so.0
```

### 7. /lost+found

- 用途：嵌入式开发，存储**丢失碎片文件**
- 场景：拔卡异常时，系统暂存丢失数据

### 8. /media

- 功能：**自动挂载** U 盘 / 外设
- 对比 Windows：相当于新增盘符

```
cd /media          //进入外设自动挂载目录
ls                 //查看U盘内容
```

### 9. /mnt

- 功能：**手动挂载** 外部设备
- 命令：mount 挂载到此目录
- 与 media 区别：media 自动，mnt 手动

### 10. /opt

- 用途：**第三方大型软件**安装目录
- 示例：Oracle、MySQL 安装位

```
cd /opt            //进入第三方软件目录
ls                 //查看安装的软件
```

### 11. /proc

- 本质：**虚拟文件系统**
- 内容：进程、内存、硬件、密钥缓存
- 特点：动态变化，不占实际磁盘

### 12. /root（root 用户家目录）**

- 路径：/root
- 权限：仅 root 可访问

```
sudo su            //切换root
cd /root           //进入root家目录
pwd                //结果 /root
exit               //退出root
```

### 13. /run

- 用途：系统运行时**临时文件**
- 示例：MySQL、Redis 运行文件

### 14. /sbin（超级用户命令）

- 含义：super binary
- 权限：**只有 root 能执行**
- 典型命令：fdisk、ifconfig、reboot、shutdown

```
cd /sbin           //进入管理员命令目录
ls                 //查看绿色可执行文件
fdisk -l                //查看磁盘分区信息（需root）//结果：显示 /dev/sda 分区列表
ifconfig                //查看/配置网卡信息  //结果：显示IP、子网掩码、MAC地址
reboot                  //重启系统     //结果：系统立即重启
shutdown -h now         //立即关机   //结果：系统立即关闭
```

bin vs sbin

- /bin：所有用户可用（ls、date、cat）
- /sbin：root 专用（磁盘、网络、关机）

### 15. /srv

- 用途：服务器服务数据（FTP、Web）

### 16. /sys

- 用途：内核、硬件底层信息

### 17. /tmp**

- 用途：**临时文件**，重启清空

```
cd /tmp            //进入临时文件目录
touch test.txt     //创建临时文件
```

### 18. /usr（Unix Software Resource）**

- 用途：安装软件、用户资源
- 重要子目录：
  - /usr/include：C/C++ 头文件（stdio.h）
  - /usr/lib：用户库文件
  - /usr/local：本地安装软件

### 19. /var（variable 变化数据）**

- 用途：存放**经常变更的数据**
- /var/log：系统日志（.log）
- /var/lib：数据库（MySQL 默认位置）
- /var/cache：应用缓存

```
cd /var/log        //进入日志目录
ls                 //查看 .log 文件
```

# Linux 文件与目录的操作命令

## ls 命令

1. 基础用法

- `ls`：仅显示当前目录下的文件 / 目录名称
- `ls 文件名/目录名`：查看指定对象

2. 常用参数 `-l`（长格式详细信息）

- `ls -l`：查看当前目录详细信息
- `ls -l 文件名`：查看指定文件详细信息

3. `ls -l` 输出 7 部分结构

   | 序号 | 内容        | 说明                              |
   | ---- | ----------- | --------------------------------- |
   | 1    | 权限 + 类型 | 文件类型 + 所有者 / 组 / 其他权限 |
   | 2    | 硬链接数    | 有多少个名字指向该文件            |
   | 3    | 所有者      | 文件属于哪个用户                  |
   | 4    | 所属组      | 文件属于哪个用户组                |
   | 5    | 文件大小    | 字节数                            |
   | 6    | 修改时间    | 最后一次修改日期 + 时间           |
   | 7    | 文件名      | 文件 / 目录名称                   |

```
-rw-r--r-- 1 root root 1024 Apr 18 10:00 test.txt
```

------

### Linux 7 种文件类型

| 类型         | 标识 | 说明                          | 存储特性       | 典型位置 / 示例          |
| :----------- | :--- | :---------------------------- | :------------- | :----------------------- |
| 普通文件     | `-`  | 文本、二进制、压缩包等        | 占用磁盘       | 随处可见                 |
| 目录文件     | `d`  | 文件夹，蓝色显示              | 占用磁盘       | 大小固定 4KB             |
| 软连接文件   | `l`  | 类似 Windows 快捷方式，淡蓝色 | 占用磁盘       | 指向真实文件路径         |
| 管道文件     | `p`  | 进程间通信，`mkfifo` 创建     | 伪文件不占空间 |                          |
| 套接字文件   | `s`  | 网络 / 本地进程通信           | 伪文件不占空间 | `/tmp` 下常见            |
| 字符设备文件 | `c`  | 键盘、终端等按字符操作设备    | 伪文件不占空间 | `/dev/tty*`              |
| 块设备文件   | `b`  | 硬盘、分区等按块操作存储设备  | 伪文件不占空间 | `/dev/sda`、`/dev/loop*` |

> 说明：后 4 种（p、s、c、b）为**伪文件**，不占用磁盘存储空间。

### 目录操作核心命令

1. `cd` 切换目录

- `cd 绝对路径`：从 `/` 开始，如 `cd /home`
- `cd 相对路径`：从当前位置跳转
- `cd ..`：返回上一级目录
- `cd .`： staying in current directory（无变化）
- cd ./test    # 进入当前目录下的test文件夹
- `cd` 或 `cd ~`：回到当前用户家目录
- `cd -`：回到上一次所在目录

2. `pwd` 查看当前路径

- 全称：print working directory
- 功能：打印当前所在工作目录绝对路径

### 文件访问权限

1、3 个权限符号含义

| 符号  | 英文    | 含义       | 对文件             | 对目录             |
| :---- | :------ | :--------- | :----------------- | :----------------- |
| **r** | read    | 读权限     | 可以查看文件内容   | 可以列出目录内容   |
| **w** | write   | 写权限     | 可以修改、删除文件 | 可以创建、删除文件 |
| **x** | execute | 执行权限   | 可以运行文件       | 可以进入目录       |
| **-** | 无      | 没有该权限 | -                  | -                  |

```
rw- r-- r--
```

1. **rw-**：所有者u → 可读 + 可写 **不能执行**

2. **r--**：所属组g → **只能读**

3. **r--**：其他用户o→ **只能读**

6. 大小 

   1.普通文件为实际大小 

   2.目录文件，是目录所占磁盘的空间大小为4k的倍数

### ls 命令其rwx 目录/文件权限含义对比

| 权限         | 对**文件**的作用                           | 对**目录**的作用                               |
| :----------- | :----------------------------------------- | :--------------------------------------------- |
| **r (读)**   | 可以**查看文件内容**cat、less、more        | 可以**列出目录里的文件名**ls                   |
| **w (写)**   | 可以**修改、删除文件内容**vim、echo、>、>> | 可以**在目录内增删改文件**touch、mkdir、rm、mv |
| **x (执行)** | 可以**运行这个文件**./ 脚本、./ 程序       | 可以**进入目录**cd、访问目录内文件             |

### ls命令的其他参数

| 参数 / 命令 | 功能说明                   | 示例            | 补充要点                                               |
| :---------- | :------------------------- | :-------------- | :----------------------------------------------------- |
| `-a`        | 显示所有文件（含隐藏文件） | `ls -a`         | 隐藏文件以 `.` 开头；包含 `.` 当前目录、`..` 上级目录  |
| `-l`        | 以长格式显示文件详细信息   | `ls -l`         | 显示权限、属主、大小、时间等                           |
| `-R`        | 递归显示所有子目录内容     | `ls -R`         | 大写 R，逐层遍历子目录                                 |
| `-h`        | 人类可读格式显示文件大小   | `ls -lh`        | 配合 `-l` 使用，显示 K/M/G 单位                        |
| `-d`        | 仅显示目录本身，不显示内容 | `ls -ld 目录名` | 查看目录自身属性                                       |
| `-F`        | 给文件添加类型标识         | `ls -F`         | 目录 `/`、可执行 `*`、软链接 `@`                       |
| `ll`        | 等同于 ls -l -a或ls -la    | `ll`            | 显示所有文件的详细信息(含隐藏文件)                     |
| `la`        | 等同于 `ls -a`             | `la`            | 显示**所有文件（包含隐藏文件）**，但**不显示详细信息** |

## cat /more/less /tail/head 命令

**Linux 系统中验证新用户是否创建成功的核心方法**，核心结论如下：

1. **目录验证**：`/home` 目录下会自动生成与用户名同名的宿主目录，用于存储用户的个人文件。
2. **登录验证**：可直接在图形界面使用新创建的用户账号登录系统，说明账号已具备登录权限。
3. 配置文件验证
   - `/etc/passwd` 文件末尾会新增该用户的关键信息，包括用户名、UID、GID、宿主目录、默认登录 Shell 等。
   - `/etc/group` 文件中会新增该用户对应的同名主组信息。

## 2.修改文件所有者 & 所属组

| :----------------------- | :-------------------------- | :-------------------------- |

| `chgrp 组名 文件`        | 只修改**所属组**            | `chgrp dev file.txt`        |
| `chown -R ...`           | 递归修改目录下所有内容      | `chown -R tom:dev testdir/` |

**小说明:**

- `-R` 大写，对目录生效，子文件 / 子目录一起改
- `:` 也可以写成 `.`：`chown tom.dev file.txt`

## 1、find 命令：**查找文件**（找文件本身）

语法：find 搜索路径（相对或者绝对）参数 条件

作用：在系统里**搜索文件 / 目录**（按名字、大小、类型、所有者查找）

| 命令格式                             | 功能                             |
| :----------------------------------- | :------------------------------- |
| `find 路径 -name "文件名"`           | **按文件名查找**（最常用）       |
| `find 路径 -type f`                  | 只找普通文件                     |
| `find 路径 -type d`                  | 只找目录                         |
| `find 路径 size +10M`                | 找大于 10M 的文件                |
| `find 路径 -user 用户名`             | 找某个用户的文件                 |
| `find 路径 -mtime -7`                | 找 7 天内修改过的文件            |
| `find 路径 -maxdepth 1 -name "file*" | 指定层级搜索目录下的所有符合文件 |
|                                      |                                  |

**最常用示例**

```bash
###########按名字######
find ./ -name "test.txt"      # ./当前目录精确查找
find /home -name "test.txt"   # 精确查找
find /home -name "*.txt"      # 模糊查找 通配符*匹配0-n任意字符   通配符？匹配一个任意字符
#########按类型##########
find . -type d                # 在当前目录找目录
find . -type f				  # 在当前目录找普通文件
find . -type P			      # 在当前目录找管道文件
find ./ -size +3M -size -5M  # 寻找大于3m小于5M为大写  k为小写 G 必须大写
##########混合参数########
find ~/ -name "*.jpg" -size +1k -size -2M  # 寻找大于1K小于2M为大写  k为小写 G 必须大写
```

------

### 1、**-exec完整固定语法**

```
find 路径 [查找条件]  -exec 命令 {} \;
find 路径 [查找条件]  -ok 命令 {} \; #询问是否执行后续命令
```

**-exec最经典示例（带注释）**

```BASH
find /home -name "*.log" -exec grep -n "error" {} \;
#- 在 `/home` 查找所有 `.log` 文件
#- 对**每个文件**执行：`grep -n "error" 文件名`
#- `{}` 自动替换成找到的文件名
```

**-exec常用实战例子**

```bash
# 1. 查找文件并显示详细信息
find . -name "*.txt" -exec ls -lh {} \;

# 2. 查找文件并删除（慎用）
find . -name "*.tmp" -exec rm -f {} \;

# 3. 查找文件并修改权限
find . -type f -exec chmod 644 {} \;

# 4. 查找并显示内容
find . -name "*.conf" -exec cat {} \;
```

### 2、`| xargs` 完整固定语法

1.**xargs基础语法**

```bash
find /home -name "*.log" | xargs grep -n "error"
# ①       ②           ③   ④     ⑤        ⑥
# ① find          查找文件命令
# ② /home         查找路径
# ③ -name "*.log" 条件：找.log文件
# ④ |             管道：把前面的 文件名 传给后面
# ⑤ xargs         接收文件名，交给后面的命令使用
# ⑥ grep -n ...   对所有找到的文件执行搜索
```

2.**xargs最常用实战**

```bash
find . -name "*.txt" | xargs ls -lh
# 查找txt文件 → 用ls -lh显示详情

find . -name "*.tmp" | xargs rm -f
# 查找临时文件 → 直接删除
```

## 2、grep 命令：**查找内容**（在文件里找文字）

作用：在**文件内容**中搜索指定字符串，并显示匹配

| 命令格式                  | 功能                         |
| :------------------------ | :--------------------------- |
| `grep "关键词" 文件名`    | 在文件中查找关键词           |
| `grep -i "关键词" 文件名` | **忽略大小写**查找           |
| `grep -n "关键词" 文件名` | 显示**行号**                 |
| `grep -r "关键词" 目录/`  | **递归**在目录所有文件里查找 |
| `grep -v "关键词" 文件名` | 反向查找（显示不匹配的行）   |

**最常用示例**

```bash
grep "root" /etc/passwd
grep -n "error" log.txt
grep -r "hello" /home/test/
```

## 两个命令**联合用法**（超级常用）

```
find /home -name "*.log" -exec grep -n "error" {} \;
```

功能：在 `/home` 下找到所有 `.log` 文件，并在这些文件里查找 `error`

## find + grep 管道用法

```bash
# 1. 基础用法：查找 .txt 文件并搜索内容 hello
find . -name "*.txt" | xargs grep "hello"
#  find .            从当前目录开始查找
#  -name "*.txt"     找后缀为 txt 的文件
#  |                 管道：把前面查到的文件名传给后面
#  xargs             将文件名转为 grep 可识别的参数
#  grep "hello"      在这些文件里搜索 hello 文本

# 2. 显示匹配内容所在行号
find . -name "*.log" | xargs grep -n "error"
#  -n    显示行号

# 3. 忽略大小写搜索
find /etc -name "*.conf" | xargs grep -i "port"
#  -i    不区分大小写

# 4. 查找所有文件并过滤含指定内容的文件
find . -type f | xargs grep -l "test"
#  -l    只显示匹配到内容的文件名
```

- `-v`：显示过程
- `-f`：指定文件名（必须放最后）

# VIM编辑器

### VIM 3 种工作模式

**1. 命令模式（Command Mode）**

- **刚打开文件时进入**
- 只能**移动光标、复制、粘贴、删除、查找**
- 不能直接输入文字
- **按 `i` 进入编辑模式**

**2. 编辑模式（Insert Mode）**

- **可以输入、修改文字**
- 屏幕左下角显示 `-- INSERT --`
- **按 `Esc` 回到命令模式**

**3. 末行模式（Last Line Mode）**

- **保存、退出、设置行号、替换**
- **在命令模式下按 `:` 进入** **末行模式**
- **连按两下esc回到命令模式**
- 常用：
  - `:w`  保存
  - `:q`  退出
  - `:wq` 保存并退出
  - `:q!` 强制退出不保存
  - `:x`保存并退出

| 8    | 系统管理命令（root 常用）     | `mount`、`ifconfig`、`iptables`    |
| 9    | 内核例程（非标准）            | 内核开发相关接口                   |

## 系统调用和库函数之前的调用关系

| 对比项   | 库函数（如 printf）        | 系统调用（如 write）           |
| :------- | :------------------------- | :----------------------------- |
| 运行态   | 用户态                     | 内核态                         |
| 依赖     | 依赖 libc 等用户态库       | 依赖操作系统内核               |
| 切换开销 | 无（用户态内执行）         | 有（用户态→内核态切换，耗时）  |
| 可移植性 | 高（标准库函数跨平台兼容） | 低（不同 OS 系统调用接口不同） |
| 关系     | 大多封装了系统调用         | 内核提供的底层接口             |

## open和close系统调用

### 一、`open` 系统调用

### 1. 核心作用

打开或创建一个文件，返回一个**文件描述符（int 类型）**，后续读写操作都通过这个描述符进行。

### 2. 函数原型

```c++
#include <sys/types.h>
#include <sys/stat.h>
#include <fcntl.h>

int open(const char *pathname, int flags, mode_t mode);
/*pathname：要打开 / 创建的文件路径（如 "./test.txt"）
flags：打开方式（常用组合）
O_RDONLY：只读打开
O_WRONLY：只写打开
O_RDWR：读写打开
O_CREAT：文件不存在时创建（必须搭配mode）
O_APPEND：追加写入
O_TRUNC：文件存在时清空内容
mode：创建文件时的权限（如 0644 表示 rw-r--r--，仅O_CREAT时有效）
*/
```

### 二、`close` 系统调用

### 1. 核心作用

关闭已打开的文件描述符，释放系统资源，防止文件句柄泄漏。

### 2. 函数原型

```c++
#include <unistd.h>

int close(int fd);
//`fd`：`open`返回的文件描述符
```

### 三、示例

```c++
#include <iostream>
#include <fcntl.h>    // Linux open 系统调用头文件
#include <unistd.h>   // Linux close 系统调用头文件
#include <cstdio>     // perror 函数，打印系统错误信息

int main()
{
    // ======================
    // open 系统调用（Linux）
    // ======================
    // 参数1：要操作的文件路径（字符串常量）
    const char* pathname = "./linux_test.txt";
    // 参数2：打开模式（只写 | 不存在则创建 | 存在则清空）
    int flags = O_WRONLY | O_CREAT | O_TRUNC;
    // 参数3：文件权限（Linux 八进制权限，0644 = rw-r--r--）
    mode_t mode = 0644;

    // 调用 open 系统调用，返回文件描述符 fd
    int fd = open(pathname, flags, mode);

    // 判断返回值：成功返回非负整数（文件描述符），失败返回 -1
    if (fd == -1)
    {
        // Linux 下用 perror 打印具体错误原因（如权限不足、路径不存在）
        perror("open 调用失败");
        return 1;
    }
    std::cout << "open 调用成功，文件描述符：" << fd << std::endl;

    // ======================
    // close 系统调用（Linux）
    // ======================
    // 参数：open 返回的文件描述符 fd
    int ret = close(fd);

    // 判断返回值：成功返回 0，失败返回 -1
    if (ret == -1)
    {
        perror("close 调用失败");
        return 1;
    }
    std::cout << "close 调用成功" << std::endl;

    return 0;
}
```

```
# 编译（使用 g++，Linux 默认已安装）
g++ linux_open_test.cpp -o linux_open_test

# 运行
./linux_open_test

# 查看生成的文件
ls -l linux_test.txt
```

------

关键说明（Linux 专属）

1. **头文件依赖**：`open`/`close` 是 Linux 系统调用，依赖 `<fcntl.h>` 和 `<unistd.h>`，标准 C++ 库中没有这些函数。
2. **权限参数**：`mode_t mode` 必须用八进制表示（如 `0644`），Linux 会根据这个值设置文件的所有者、组和其他用户的读写权限。
3. **错误处理**：Linux 下用 `perror()` 可以直接打印系统错误信息（比如 “Permission denied”“No such file or directory”），比 `cout` 更实用。
4. **文件描述符**：Linux 中 0/1/2 分别对应标准输入、标准输出、标准错误，所以 `open` 返回的文件描述符通常从 3 开始。







# 函数库（Library）

**函数库 = 预先编译好的、可复用的函数集合**

别人写好的常用代码（比如 `printf`、`sin`、`open`）打包在一起，方便你直接调用，不用自己重复写。

------

**函数库整体结构**

- 本质：一堆 **.o 目标文件** 打包而成
- 作用：代码复用、模块化、减少重复造轮子
- 分类：
  1. **静态库**（.a / .lib）
  2. **动态库**（.so / .dll）

**库的使用流程（极简版）**

1. 写代码调用库函数
2. 编译时告诉编译器用哪个库
3. 链接阶段：
   - 静态库：把用到的函数**复制进可执行文件**
   - 动态库：只记录引用，**运行时再加载**

### 静态库 vs 动态库 

这俩是 C/C++ 里两种不同的代码复用方式，核心区别在于**代码什么时候被整合进程序里**

| 对比项             | 静态库（`.a` / `.lib`）                  | 动态库（`.so` / `.dll`）                      |
| :----------------- | :--------------------------------------- | :-------------------------------------------- |
| **链接时机**       | **编译链接阶段**                         | **程序运行阶段**                              |
| **代码整合方式**   | 链接时，把库的代码直接复制到可执行文件里 | 可执行文件里只存一个 “占位符”，运行时再加载库 |
| **可执行文件大小** | 大（包含了库的全部代码）                 | 小（不包含库代码，只存引用）                  |
| **内存占用**       | 多进程运行时，每个进程都有一份库代码副本 | 多进程可共享同一份库代码，节省内存            |
| **更新方式**       | 库更新后，必须重新编译链接程序才能生效   | 直接替换动态库文件即可，程序不用重新编译      |
| **依赖关系**       | 可执行文件不依赖外部库，能独立运行       | 运行环境必须有对应的动态库文件，否则会报错    |
| **Linux 工具**     | `ar`（打包静态库）、`ld` 链接            | `gcc -shared` 编译、`ld` 动态链接             |

**静态库 vs 动态库制作**

| 步骤     | 静态库（.a）            | 动态库（.so）                    |
| :------- | :---------------------- | :------------------------------- |
| 1 编译   | `gcc -c xxx.c`          | `gcc -c -fPIC xxx.c`             |
| 2 生成库 | `ar rcs libxxx.a xxx.o` | `gcc -shared xxx.o -o libxxx.so` |
| 3 链接   | `gcc main.c -L. -lxxx`  | 同上                             |
| 4 运行   | 直接运行                | `LD_LIBRARY_PATH=. ./程序`       |

### 制作静态/动态库例子

```bash
项目根目录
├── hello.c              # 主程序源文件
├── inc/                 # 头文件目录
│     └── mymath.h       # 函数声明
├── lib/                 # 库文件目录
│     ├── libmymath.a    # 静态库
│     └── libmymath.so   # 动态库
└── src/                 # 功能源文件目录
      └── mymath.c       # 函数实现
```

**1. 先建目录，放文件**

```bash
# 建目录
mkdir -p inc lib src

# 写头文件 inc/mymath.h
cat > inc/mymath.h << 'EOF'
#ifndef MYMATH_H
#define MYMATH_H

int add(int a, int b);
int sub(int a, int b);

#endif
EOF

# 写库实现 src/mymath.c
cat > src/mymath.c << 'EOF'
#include "mymath.h"

int add(int a, int b) {
    return a + b;
}

int sub(int a, int b) {
    return a - b;
}
EOF

# 写主程序 hello.c
cat > hello.c << 'EOF'
#include <stdio.h>
#include "mymath.h"

int main() {
    printf("1+2=%d\n", add(1,2));
    printf("5-3=%d\n", sub(5,3));
    return 0;
}
EOF
```

**2. 制作静态库 libmymath.a 到 lib 目录**

```bash
# 编译成目标文件（注意 -I 指定头文件路径）
gcc -c src/mymath.c -o mymath.o -I ./inc

# 打包成静态库，放到 lib 目录
ar rcs lib/libmymath.a mymath.o
```

**3. 用你这条命令编译主程序（链接静态库）**

```bash
gcc hello.c -o app_static -L ./lib -l mymath -I ./inc

# 运行
./app_static

#查看库
file lib/mymath.a
```

**4. 制作动态库 libmymath.so 到 lib 目录**

```bash
# 编译成位置无关代码
gcc -c -fPIC src/mymath.c -o mymath_pic.o -I ./inc  

# 生成动态库，放到 lib 目录
gcc -shared mymath_pic.o -o lib/libmymath.so
```

**5. 用同样的命令编译主程序（链接动态库）**

```bash
gcc hello.c -o app_dynamic -L ./lib -l mymath -I ./inc

# 运行（必须指定库路径）
LD_LIBRARY_PATH=./lib ./app_dynamic

#查看库
file lib/libmymath.so
```





# Makefile 知识点

| 类别             | 名称 / 写法     | 作用                                          | 示例                                   |
| :--------------- | :-------------- | :-------------------------------------------- | :------------------------------------- |
| **1 个规则**     | 目标：依赖 命令 | Make 基本结构：目标要靠依赖生成，然后执行命令 | `app:hello.c` <br />      gcc $< -o $@ |
| **2 个函数**     | `wildcard`      | **自动遍历文件**，匹配路径下所有 .c           | `src=$(wildcard src/*.c)`              |
|                  | `patsubst`      | **批量替换后缀**，把 .c 换成 .o               | `obj=$(patsubst %.c,%.o,$src)`         |
| **3 个自动变量** | `$@`            | 代表**目标**                                  | `gcc ... -o $@`                        |
|                  | `$<`            | 代表**第一个依赖**                            | `gcc $< -o ...`                        |
|                  | `$^`            | 代表**所有依赖**                              | `gcc $^ -o ...`                        |

### makefile

- 作用：**项目编译管理**，一键编译，不用每次敲长串 gcc 命令。
- 初步学习：**1 个规则、2 个函数、3 个自动变量**。
- 文件名要求：必须是 `makefile` 或 `Makefile`，才能直接用默认的 `make` 命令。

**语法：**

```bash
目标: 依赖条件
	命令  # 必须用Tab缩进，不能用空格
hello：hello.c
	gcc hello.c -o hello #用 gcc 把 hello.c 编译成可执行文件，名字叫 hello。
```

**核心逻辑**

- 时间检查：目标文件的修改时间，必须晚于所有依赖文件的修改时间；否则会重新执行命令，更新目标。
- 依赖生成：如果依赖条件不存在，make 会自动寻找能生成它的其他规则，递归构建。

### 极简可运行 Makefile

```makefile
# 定义编译器为 gcc
CC := gcc

# 1. 自动找到当前目录下所有 .c 文件
SRC := $(wildcard *.c)

# 2. 把所有 .c 文件替换成 .o 文件
OBJ := $(patsubst %.c, %.o, $(SRC))

# 总目标：最终要生成的程序叫 app
all: app

# 3. 链接所有 .o 文件，生成可执行文件 app
# $^ 代表所有依赖（所有 .o）
# $@ 代表目标（app）
app: $(OBJ)
	$(CC) $^ -o $@

# 4. 模式规则：把每个 .c 编译成 .o
# $< 代表第一个依赖（对应的 .c 文件）
%.o: %.c
	$(CC) -c $< -o $@

# 清理：删除所有 .o 和可执行文件
clean:
	rm -f *.o app
```

### 模式规则 vs 静态模式规则

| 名称             | 写法               | 作用范围          | 特点       |
| :--------------- | :----------------- | :---------------- | :--------- |
| **模式规则**     | `%.o: %.c`         | 所有 `.o`         | 通用、简单 |
| **静态模式规则** | `$(OBJ): %.o: %.c` | 只给变量里的 `.o` | 精准、安全 |

##### **声明伪目标** ﻿

- 实现步骤
  - 在makefile中添加.PHONY: clean声明
  - 多个伪目标可用空格分隔，如.PHONY: clean all
