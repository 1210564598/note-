# 第九章复合类型（自定义类型）

## 9.1 结构体

### 9.1.1 概述

结构体是 C 语言**构造数据类型**，用于将**不同类型的数据**组合成一个有机整体（如学生的学号、姓名、年龄等属性），解决数组只能存放相同类型数据的问题，便于数据管理。

### 9.1.2 结构体变量的定义和初始化

结构体**类型**是数据模型，不分配内存；结构体**变量**基于类型创建，系统为其分配实际内存，有 3 种定义方式：

1. **先声明类型，再定义变量（常用）**

```c
// 声明结构体类型
struct stu {
    char name[50];
    int age;
};
// 定义变量并初始化
struct stu s1 = { "mike", 18 };
```

1. **声明类型同时定义变量**

```c
struct stu2 {
    char name[50];
    int age;
}s2 = { "lily", 22 }; // 直接定义变量s2
```

1. **直接定义变量（无类型名，仅能使用一次）**

```c
struct {
    char name[50];
    int age;
}s3 = { "yuri", 25 }; // 无类型名，无法复用
```

### 9.1.3 结构体成员的访问

- **普通结构体变量**：使用**点运算符 `.`** 访问成员
- **结构体指针变量**：使用**箭头运算符 `->`** 访问成员（等价于`(*指针).成员`）

```c
#include<stdio.h>
#include<string.h>
struct stu { char name[50]; int age; };
int main() {
    struct stu s1;
    // 普通变量访问
    strcpy(s1.name, "abc");
    s1.age = 18;
    // 指针访问（取地址后用->）
    strcpy((&s1)->name, "test");
    (&s1)->age = 22;
    return 0;
}
```

### 9.1.4 结构体数组

数组元素为结构体类型，用于存储一组同类型的结构体数据（如多个学生信息），通过**数组下标 +`.`** 访问成员，也可通过`数组名+偏移量->`访问。

```c
struct stu boy[5] = { {101, "Li ping", 'M', 45}, {102, "Zhang ping", 'M', 62.5} };
// 访问方式1：下标+点运算符
boy[i].score;
// 访问方式2：地址+箭头运算符
(boy+i)->score;
```

### 9.1.5 结构体嵌套

一个结构体的**成员**是另一个已定义的结构体类型，访问嵌套成员需**逐层使用`.`** （普通变量）或`->`（指针变量）。

```c
// 内层结构体
struct person { char name[20]; char sex; };
// 外层结构体嵌套person
struct stu { int id; struct person info; };
// 访问嵌套成员：逐层访问
struct stu s[2] = { 1, "lily", 'F', 2, "yuri", 'M' };
s[i].info.name; // 访问stu的info成员的name
```

### 9.1.6 结构体赋值

**相同类型的结构体变量**可直接相互赋值，系统会将源变量的**所有成员值逐字节拷贝**到目标变量，二者内存独立，修改其一不影响另一个（等价于`memcpy`拷贝）。

```c
struct stu s1 = { "abc", 18 };
struct stu s2 = s1; // 直接赋值，所有成员值拷贝
```

### 9.1.7 结构体和指针

#### 1) 指向普通结构体变量的指针

定义结构体指针，指向结构体变量的地址，通过`->`访问成员，`(*p).成员`与`p->成员`等价。

```c
struct stu s1 = { "lily", 18 };
struct stu *p = &s1;
p->name; // 等价于(*p).name
```

#### 2) 堆区结构体变量

通过`malloc`在堆区分配结构体内存，返回值强制转换为结构体指针，使用后需**手动 free 释放**，避免内存泄漏。

```c
struct stu *p = (struct stu *)malloc(sizeof(struct stu));
strcpy(p->name, "test"); // 堆区变量访问
free(p); p = NULL; // 释放并置空，避免野指针
```

#### 3) 结构体嵌套一级指针

若结构体成员包含指针，需**分两步分配内存**：先分配结构体本身的堆内存，再为指针成员分配堆内存；释放时**先释放指针成员，再释放结构体**，避免内存泄漏。

```c
struct stu { char *name; int age; }; // 成员为一级指针
struct stu *p = (struct stu *)malloc(sizeof(struct stu));
p->name = malloc(strlen("test") + 1); // 为指针成员分配内存
strcpy(p->name, "test");
// 释放顺序：先成员，后结构体
free(p->name); p->name = NULL;
free(p); p = NULL;
```

### 9.1.8 结构体做函数参数

结构体传参有**值传递**和**地址传递**两种方式，地址传递效率更高（仅传 4/8 字节地址，而非整个结构体数据），是实际开发的首选。

#### 1) 普通变量做参数（值传递）

函数接收结构体副本，**修改形参不会影响实参**，适合小型结构体，大型结构体易造成栈溢出。

```c
void set_stu(struct stu tmp) { strcpy(tmp.name, "mike"); } // 形参是副本
int main() { struct stu s = {0}; set_stu(s); } // 实参值不变
```

#### 2) 指针变量做参数（地址传递）

函数接收结构体地址，通过指针**直接操作实参的内存**，修改形参会同步影响实参，效率高。

```c
void set_stu_pro(struct stu *tmp) { strcpy(tmp->name, "mike"); }
int main() { struct stu s = {0}; set_stu_pro(&s); } // 实参被修改
```

#### 3) 结构体数组名做参数

数组名本质是首元素地址，传参时等价于结构体指针，函数内可通过指针遍历数组，修改数组元素的成员。

```c
void set_stu_pro(struct stu *tmp, int n) { // 数组名传参=指针
    for (int i=0; i<n; i++) { tmp->age = 20+i; tmp++; }
}
int main() { struct stu s[3] = {0}; set_stu_pro(s, 3); }
```

#### 4) const 修饰结构体指针形参

用于**保护实参不被修改**，根据`const`位置不同，保护效果不同，开发中常用于只读操作的函数形参：

- `struct stu * const p`：指针**指向不可变**，但指向的结构体成员**可修改**
- `const struct stu * p`：指针**指向可变**，但指向的结构体成员**不可修改**（常用）
- `const struct stu * const p`：指针**指向不可变**，成员也**不可修改**

## 9.2 共用体（联合体）

### 核心特性

1. 共用体是**同一块内存空间**存储不同类型数据的构造类型，所有成员**共享同一块内存**；
2. 共用体的**内存大小 = 最长成员的内存大小**（内存对齐）；
3. 同一时刻**只有一个成员有效**，新成员赋值会**覆盖原有成员的值**；
4. 共用体变量的地址和其**所有成员的地址完全相同**。

### 示例代码

```c
union Test {
    unsigned char a; // 1字节
    unsigned int b;  // 4字节
    unsigned short c;// 2字节
}; // 共用体大小为4字节（最长成员b）
int main() {
    union Test tmp;
    printf("%p, %p, %p\n", &tmp.a, &tmp.b, &tmp.c); // 地址完全相同
    tmp.b = 0x44332211; // 赋值int，覆盖所有成员
    printf("%x\n", tmp.a); // 11（低1字节）
    printf("%x\n", tmp.c); // 2211（低2字节）
    return 0;
}
```

**内存存储规则**：低字节存低地址，高字节存高地址。

## 9.3 枚举

### 核心特性

1. 枚举是将变量的**所有可能值一一列举**，变量值**仅限于列举的范围**，用于表示有限的状态（如星期、真假、颜色）；
2. 枚举值是**常量**，**不能在程序中赋值修改**；
3. 枚举元素的默认值是**从 0 开始的连续整数**，也可手动指定初始值，后续元素依次递增。

### 定义和使用

```c
// 手动指定sun=2，后续mon=3、tue=4...
enum weekday { sun = 2, mon, tue, wed, thu, fri, sat };
// 默认flase=0，true=1
enum bool { flase, true };
int main() {
    enum weekday a = sun; // 枚举变量只能赋值枚举元素
    enum bool flag = true;
    printf("%d, %d\n", a, flag); // 输出2, 1
    return 0;
}
```

## 9.4 typedef

### 核心作用

`typedef`是 C 语言关键字，用于**为已有数据类型定义一个新名字**（别名），**不能创建新类型**，仅简化类型书写，提高代码可读性和可移植性。

### 核心区别（与 #define）

| 特性     | typedef            | #define                |
| :------- | :----------------- | :--------------------- |
| 生效阶段 | 编译阶段           | 预处理阶段             |
| 作用范围 | 仅针对**数据类型** | 可定义常量、宏、别名等 |
| 语法要求 | 末尾必须加**分号** | 末尾不加分号           |
| 类型检查 | 编译器会做类型检查 | 简单字符串替换，无检查 |

### 使用场景

1.为基本类型定义别名

```c
typedef int INT;        // INT等价于int
typedef unsigned char UBYTE; // UBYTE等价于unsigned char
```

2.为自定义类型（结构体、枚举、共用体）定义别名，简化书写

```c
// 定义结构体并取别名TYPE，指针别名PTYPE
typedef struct type {
    UBYTE a;
    INT b;
}TYPE, *PTYPE;
// 直接用别名定义变量，无需写struct
TYPE t;
PTYPE p = &t; // 等价于struct type *p = &t;
```
