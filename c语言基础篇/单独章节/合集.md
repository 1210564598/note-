# 第一章C 语言概述

## 一、C 语言概述

1.本质：人与计算机的交流语言，遵循语法规则，计算机无条件服从。

2.特点：优点为代码量小、执行快、功能强、编程自由；缺点为开发周期长、可移植性差、易出错、依赖平台库。

3.学习价值：理工科必修课、面试必备，是底层开发及其他语言（Java/C++ 等）的基础。

4.应用领域：操作系统 / 驱动、嵌入式、高性能服务（Apache/Nginx）、数据库、编译器等。

5.简洁性：32 个关键字、9 种控制语句、34 种运算符，可实现复杂功能。

## 四、C 语言编译过程

四步流程：

预处理：展开头文件、宏定义，删除注释，不检查语法。

编译：检查语法，生成汇编文件（.s）。

汇编：生成二进制目标文件（.o）。

链接：链接库文件，生成可执行文件。

分步编译命令：

预处理：gcc -E hello.c -o hello.i。

编译：gcc -S hello.i -o hello.s。

汇编：gcc -c hello.s -o hello.o。

链接：gcc hello.o -o hello。

# 第二章数据类型

## **一、常量与变量**

### 1. 核心概念

- 常量：程序运行中值不可改，分为整型（100、-200）、实型（3.14、0.125）、字符型（'a'、'\\n'）、字符串型（"abc"）；宏定义常量（#define MAX 10）、const 常量（const int b=10）均不可修改。
- 变量：程序运行中值可变，需先定义后使用（格式：类型 变量名），编译时分配内存，可通过名字或地址访问。

## 2. 变量命名规则

- 不能是关键字（如 int、char）；
- 由字母、数字、下划线组成，首字符必须是字母或下划线；
- 区分大小写（Age≠age）。

### 3. 声明与定义区别

- 声明：不分配存储空间（如extern int a），仅告知变量存在；
- 定义：分配存储空间（如int b=10），兼具声明作用。

## **二、整型（int）**

### 1. 整型分类与占用空间

| **类型**        | **占用空间**                   | **取值范围**                             |
| --------------- | ------------------------------ | ---------------------------------------- |
| short（短整型） | 2 字节                         | -32768~32767                             |
| int（整型）     | 4 字节                         | -2147483648~2147483647                   |
| long（长整型）  | Windows=4 字节；Linux=4/8 字节 | 同 int 或更大范围                        |
| long long       | 8 字节                         | -9223372036854775808~9223372036854775807 |
| unsigned 修饰   | 同对应类型                     | 非负数（如 unsigned int：0~4294967295）  |

### 2. 赋值与输入输出

- 赋值格式：十进制（123）、八进制（0123）、十六进制（0x123）；
- 输出格式：%d（有符号十进制）、%o（八进制）、%x/%X（十六进制）、%u（无符号十进制）；
- 输入格式：scanf("%d", &变量名)（& 为取地址符，不可省略）。

### 3. 有符号与无符号区别

- 有符号数：最高位为符号位（0 = 正，1 = 负），可存正负值；
- 无符号数：最高位为数值位，仅存非负数，取值范围翻倍。

## **三、sizeof 关键字**

- 功能：计算数据类型 / 变量的占用字节数，非函数，无需头文件；
- 返回值：size_t 类型（32 位为 unsigned int）；
- 示例：sizeof(int)=4、sizeof(char)=1。

## **四、字符型（char）**

- 1. 核心特性
- 占用 1 字节，本质是 1 字节整型，存储字符的 ASCII 编码；
- 赋值需用单引号（char ch='a'），不可用双引号。
- 2. 输入输出与转换
- 输出：%c（打印字符）、%d（打印 ASCII 码）；
- 大小写转换：'a'-32='A'、'A'+32='a'（空格 ASCII 码为 32，也可通过'a'-' '转换）。
- 3. 转义字符（常用）

| -   转义字符 | -   含义         | -   ASCII 码 |
| ------------ | ---------------- | ------------ |
| -   \\n      | -   换行         | -   010      |
| -   \\t      | -   水平制表     | -   009      |
| -   \\r      | -   回车         | -   013      |
| -   \\       | -   反斜线       | -   092      |
| -   '        | -   单引号       | -   039      |
| -   "        | -   双引号       | -   034      |
| -   \\0      | -   字符串结束符 | -   000      |

## **五、实型（浮点型）**

- 1. 分类与特性

| -   类型   | -   占用空间 | -   有效数字 | -   赋值格式             |
| ---------- | ------------ | ------------ | ------------------------ |
| -   float  | -   4 字节   | -   6~7 位   | -   需加 f 后缀（3.14f） |
| -   double | -   8 字节   | -   15~16 位 | -   默认浮点型（3.14）   |

- 2. 赋值与输出
- 科学计数法：3.2e3f=3200、100e-3f=0.1；
- 输出：%f（float/double）、%lf（double），默认保留 6 位小数。

## **六、进制**

- 1. 进制表示与转换

| -   进制     | -   C 语言表示 | -   转换方法                               |
| ------------ | -------------- | ------------------------------------------ |
| -   十进制   | -   123        | -   直接书写                               |
| -   八进制   | -   0123       | -   十进制数 ÷8，余数倒序排列              |
| -   十六进制 | -   0x123      | -   十进制数 ÷16，余数倒序（10~15 用 A~F） |
| -   二进制   | -   无直接表示 | -   十进制数 ÷2，余数倒序排列              |

- 2. 常用单位
- 1bit（位）=1 二进制位；1Byte（字节）=8bit；
- 1KB=1024Byte，1MB=1024KB，1GB=1024MB，1TB=1024GB。

## **七、内存数值存储方式**

- 1. 原码、反码、补码
- 正数：三者相同（符号位 0，数值位为绝对值二进制）；
- 负数：原码（符号位 1，数值位为绝对值二进制）→ 反码（符号位不变，数值位取反）→ 补码（反码 + 1）；
- 计算机统一用补码存储，解决 0 的二义性（+0 和 - 0 补码均为 0000 0000），将减法转为加法。
- 2. 数值溢出
- 有符号数：溢出后正负反转（如 char 型 127+2=-127）；
- 无符号数：溢出后高位丢失（如 unsigned char 型 255+1=0）。

## **八、类型限定符**

| -   限定符   | -   含义                                   |
| ------------ | ------------------------------------------ |
| -   extern   | -   声明变量，不分配存储空间               |
| -   const    | -   定义常量，值不可修改                   |
| -   volatile | -   防止编译器优化代码（如硬件寄存器变量） |
| -   register | -   建议将变量存入 CPU 寄存器，提高效率    |

## **九、字符串格式化输入输出**

- 1. 字符串常量
- 以'\\0'为结束符（编译器自动添加），如 "a" 占用 2 字节（'a'+'\\0'）。
- 2. 输出函数
- printf：支持多格式输出，常用格式符%d（int）、%c（char）、%f（浮点型）、%s（字符串）、%p（指针地址）；
- putchar：仅输出单个字符（putchar('a')）。
- 3. 输入函数
- scanf：输入时格式符需与变量类型匹配，如scanf("%c", &ch)、scanf("%d", &a)；
- getchar：读取单个字符，可用于吸收输入缓冲区的回车符。

# 第三章 运算符与表达式

## 一、常用运算符分类

C 语言运算符按功能可分为 6 大类，核心作用各有侧重，是构成表达式的基础：

| **运算符类型** | **核心作用**                                    |
| -------------- | ----------------------------------------------- |
| 算术运算符     | 处理四则运算、自增自减、取模等数值计算          |
| 赋值运算符     | 将表达式计算结果赋给变量，含复合赋值            |
| 比较运算符     | 比较表达式大小 / 相等关系，返回真 (1)/ 假 (0)   |
| 逻辑运算符     | 对布尔值（0/1）进行逻辑判断，返回真 (1)/ 假 (0) |
| 位运算符       | 对数据的二进制位进行移位、与 / 或 / 异或等操作  |
| sizeof 运算符  | 计算数据 / 变量 / 类型占用的内存字节数          |

## 二、核心运算符详解

### （一）算术运算符

包含基础四则、取模、自增自减，****自增自减分前后，结果差异关键****，取模仅适用于整型：

| **运算符** | **术语** | **示例**   | **结果** |
| ---------- | -------- | ---------- | -------- |
| +          | 正号     | +3         | 3        |
| -          | 负号     | -3         | -3       |
| +          | 加       | 10+5       | 15       |
| -          | 减       | 10-5       | 5        |
| *          | 乘       | 10*5       | 50       |
| /          | 除       | 10/5       | 2        |
| %          | 取模     | 10%3       | 1        |
| ++         | 前自增   | a=2;b=++a; | a=3;b=3  |
| ++         | 后自增   | a=2;b=a++; | a=3;b=2  |
| --         | 前自减   | a=2;b=--a; | a=1;b=1  |
| --         | 后自减   | a=2;b=a--; | a=1;b=2  |

### （二）赋值运算符

基础赋值为=，复合赋值是基础运算+=/-=/*=/=/=%=，简化运算赋值一步操作：

| **运算符** | **术语** | **示例**  | **结果** |
| ---------- | -------- | --------- | -------- |
| =          | 赋值     | a=2;b=3;  | a=2;b=3  |
| +=         | 加等于   | a=0;a+=2; | a=2      |
| -=         | 减等于   | a=5;a-=3; | a=2      |
| *=         | 乘等于   | a=2;a*=2; | a=4      |
| /=         | 除等于   | a=4;a/=2; | a=2      |
| %=         | 模等于   | a=3;a%=2; | a=1      |

### （三）比较运算符

****C 语言中真用 1 表示，假用 0 表示****，仅判断关系，不做数值计算：

| **运算符** | **术语** | **示例** | **结果** |
| ---------- | -------- | -------- | -------- |
| ==         | 相等于   | 4==3     | 0        |
| !=         | 不等于   | 4!=3     | 1        |
| <          | 小于     | 4<3      | 0        |
| \>         | 大于     | 4>3      | 1        |
| <=         | 小于等于 | 4<=3     | 0        |
| \>=        | 大于等于 | 4>=1     | 1        |

### （四）逻辑运算符

操作数为 0（假）或非 0（真），核心是逻辑与 / 或 / 非的判断规则：

| 运算符 | 术语 | 示例 | 结果规则                   |
| ------ | ---- | ---- | -------------------------- |
| !      | 非   | !a   | a 假则！a 真，a 真则！a 假 |
| &&     | 与   | a&&b | 两者都真则真，否则假       |
| \|\    |      | 或   | a\|\|b                     |

## 三、运算符优先级与结合方向

优先级决定运算执行顺序（****数字越小，优先级越高****），结合方向决定同优先级运算的执行顺序，核心规则如下：

. 最高优先级（1）：数组下标[]、圆括号()、成员选择. / ->，左到右结合
. 单目运算符（2）：负号-、按位取反~、自增++/ 自减--、取值*、取地址&、逻辑非!、强制类型转换(类型)、sizeof，右到左结合
. 算术乘除模（3）：/ \* %，左到右结合（双目）
. 算术加减（4）：+ -，左到右结合（双目）
. 位移位（5）：<< >>，左到右结合（双目）
. 比较大小（6）：> >= < <=，左到右结合（双目）
. 比较相等（7）：== !=，左到右结合（双目）
. 按位与（8）：&，左到右结合（双目）
. 按位异或（9）：^，左到右结合（双目）
. 按位或（10）：|，左到右结合（双目）
. 逻辑与（11）：&&，左到右结合（双目）
. 逻辑或（12）：||，左到右结合（双目）
. 三目运算符（13）：条件运算符?:，右到左结合（唯一三目）
. 赋值运算符（14）：基础赋值=、复合赋值+=/-=/*=/等，右到左结合
. 最低优先级（15）：逗号运算符,，左到右结合

## 四、类型转换

不同类型数据混合运算时需进行类型转换，遵循****精度不降低****原则（值域小→值域大，字节数少→字节数多），分两种方式：

### （一）隐式转换（自动转换）

由编译系统自动完成，无需手动干预，运算数类型不同时，低精度向高精度转换。

示例代码：

```c
#include <stdio.h>
int main()
{
    int num = 5;
    printf("s1=%d\n", num / 2);    // 整型相除，结果为整型2
    printf("s2=%lf\n", num / 2.0); // 整型与浮点型运算，num自动转为浮点型，结果2.500000
    return 0;
}

```

### （二）强制类型转换

手动使用强制类型转换运算符完成，强制将表达式结果转为指定类型，语法：(类型说明符) (表达式)。注意：仅临时转换运算结果，不改变原变量的类型和值。

示例代码：

```c
#include <stdio.h>
int main()
{
    float x = 0;
    int i = 0;
    x = 3.6f;
    i = (int)x;  // 强制将浮点型3.6转为整型3，i=3
    printf("x=%f, i=%d\n", x, i); // 输出x=3.600000, i=3
    return 0;
}
```

# 第四章 程序流程结构

## 4.1 核心结构概述

C 语言支持 3 种基础程序运行结构，是程序逻辑设计的基础，所有复杂程序均由其组合构成：

. ****顺序结构****：按代码书写顺序依次执行，无跳转、无分支，是最基础的结构；
. ****选择结构****：根据条件是否满足，选择性执行对应代码块，实现逻辑分支；
. ****循环结构****：根据条件判断结果，重复执行某段代码，直到条件不满足。

## 4.2 选择结构

通过条件判断实现代码分支，包含if系列语句、三目运算符、switch语句，适配不同分支场景。

. ****if 语句（单分支）****：仅当条件为真时执行代码块，条件为假则跳过；
. ****if…else 语句（双分支）****：条件为真执行if块，为假执行else块，二者必选其一；
. ****if…else if…else 语句（多分支）****：依次判断多个条件，满足即执行对应块，后续条件不再判断，无满足条件则执行else；
. ****三目运算符****：语法条件?表达式1:表达式2，简化简单的双分支if…else，条件为真执行表达式 1，否则执行表达式 2；
. ****switch 语句（多值匹配）****：根据整型 / 字符型变量值匹配case分支，case后需加break跳出（否则穿透执行），无匹配项执行default。

## 4.3 循环结构

实现代码的重复执行，核心包含****循环条件****和****循环体****，支持嵌套使用，三种循环可相互转换。

. ****while 循环****：先判断条件，再执行循环体，条件不满足则直接跳出，****可能一次都不执行****；
. ****do…while 循环****：先执行一次循环体，再判断条件，****至少执行一次****，适合需要先执行后判断的场景；
. ****for 循环****：语法for(初始化;条件;更新)，结构最清晰，初始化仅执行一次，适合****循环次数明确****的场景，是最常用的循环；
. ****嵌套循环****：循环内部嵌套另一个 / 多个循环，外层循环执行一次，内层循环执行完整周期，常用于处理二维 / 多维数据（如矩阵、表格）。

## 4.4 跳转语句

用于改变程序默认执行流程，实现循环 / 分支的提前退出、跳过，包含break/continue/goto，使用场景各有不同。

. ****break****：在switch中终止case并跳出结构；在循环中跳出****当前 / 最近的内层循环****，嵌套循环中仅影响所在层；
. ****continue****：仅用于循环，****终止本次循环****，跳过循环体后续代码，直接进入下一次条件判断；
. ****goto****：无条件跳转到代码中指定标识位置，易导致代码可读性差、逻辑混乱，****尽量少用****，仅适用于多层循环一次性跳出等特殊场景。

# 第五章 数组和字符串

## 5.1 数组核心概述

. ****定义****：将****相同数据类型****的若干变量按****有序形式****组织起来的****构造数据类型****，在内存中占用****连续的存储空间****；
. ****核心特性****：所有元素类型相同、内存地址连续、通过****下标访问****（下标从 0 开始）；
. ****分类****：按元素类型分为数值数组、字符数组、指针数组等；按维数分为一维数组、多维数组（二维及以上，内存本质为一维连续存储）；
. ****数组名特性****：数组名是****地址常量****，代表数组首元素的地址，不可被赋值，可通过sizeof(数组名)计算数组整体内存大小。

## 5.2 一维数组

### 5.2.1 定义与使用

语法：类型说明符 数组名[常量表达式];，如int a[10];表示定义 10 个 int 类型元素的一维数组，元素为a[0]~a[9]（无a[10]）。

- 数组名符合标识符规则，同一作用域内唯一；
- 定义时[]内为****常量****，使用时[]内可为常量 / 变量；
- 遍历常用for循环，通过下标访问每个元素。

### 5.2.2 初始化

定义数组的同时赋值，未手动初始化的数组有默认值：****全局数组默认 0，局部数组为随机值****。常见方式：

- 全初始化：int a[10] = {1,2,3,...,10};
- 部分初始化：int a[10] = {1,2,3};（前 3 个赋值，剩余自动补 0）
- 全置 0：int a[10] = {0};（最常用）
- 省略长度：int a[] = {1,2,3};（长度由初始化值个数决定）

### 5.2.3 常用操作（强化训练）

. ****求最值****：假设首元素为最值，遍历数组依次比较，更新最值；
. ****数组逆置****：双下标从首尾向中间遍历，交换对应位置元素；
. ****冒泡排序****：相邻元素两两比较，将较大 / 较小元素逐步 “冒泡” 到数组末尾，外层循环控制轮数，内层循环控制每轮比较次数。

## 5.3 二维数组

### 5.3.1 定义与使用

语法：类型说明符 数组名[行数][列数];，如int a[3][4];表示 3 行 4 列的二维数组，可理解为 “3 个一维数组（每个含 4 个 int 元素）组成的数组”。

- 元素访问：数组名[行下标]\[列下标]，行 / 列下标均从 0 开始；
- 内存存储：****按行优先****连续存储，先存第 0 行，再存第 1 行，依次类推；
- 遍历常用****双层 for 循环****，外层遍历行，内层遍历列。

### 5.3.2 初始化

支持****分段赋值（按行）和连续赋值（按内存顺序）****，未初始化元素自动补 0，省略行数时由初始化值决定（列数不可省）。常见方式：

- 分段赋值：int a[3][4] = {{1,2},{3,4},{5,6}};（可读性高，推荐）
- 连续赋值：int a[3][4] = {1,2,3,4,5,6};
- 全置 0：int a[3]\[4] = {0};
- 省略行数：int a[]\[4] = {1,2,3,4,5,6};（自动识别为 2 行）

### 5.3.3 数组长度计算

通过sizeof结合数组名特性计算，核心公式：

- 行数 = sizeof(数组名) / sizeof(数组名[0])
- 列数 = sizeof(数组名[0]) / sizeof(数组名[0]\[0])
- 总元素数 = sizeof(数组名) / sizeof(数组名[0]\[0])

## 5.4 多维数组（了解）

- 定义：与二维数组类似，语法类型说明符 数组名[n1]\[n2]…[nn];，如int a[3]\[4]\[5];（三维数组）；
- 理解：多维数组是 “嵌套的数组”，三维数组可理解为 “多个二维数组组成的一维数组”；
- 适用场景：极少使用，仅用于处理三维及以上多维数据，日常开发以一维 / 二维数组为主。

## 5.5 字符数组与字符串

C 语言****无专门的字符串数据类型****，通过****字符数组****模拟字符串，字符串是****以 '\\0'（数字 0，结束符）结尾的特殊字符数组****，'\\0'不占用字符串有效长度。

### 5.5.1 核心区别

. ****普通字符数组****：无'\\0'结尾，仅为字符集合，用%s输出会乱码；
. ****字符串****：以'\\0'结尾的字符数组，%s输出时遇到'\\0'停止，后续字符不输出。

### 5.5.2 字符串初始化

推荐****双引号直接赋值****，编译器会自动在末尾添加'\\0'，避免手动遗漏。常见方式：

- 普通字符数组（无'\\0'）：char c[] = {'a','b','c'};（输出乱码）
- 指定长度（自动补 0）：char c[10] = {'a','b','c'};（成为字符串）
- 双引号赋值（最常用）：char c[] = "abcdef";（自动加'\\0'）
- 全置 0：char c[100] = {0};

### 5.5.3 字符串输入输出

专用函数适配不同场景，****推荐使用安全的 fgets/fputs****，避免缓冲区溢出，核心函数对比：

| **函数**              | **功能**       | **特点 & 注意事项**                                    |
| --------------------- | -------------- | ------------------------------------------------------ |
| scanf("%s",str)       | 输入字符串     | 以空格 / 回车分隔，不读空格，易溢出                    |
| gets(str)             | 输入字符串     | 读空格，以回车分隔，极易溢出（废弃）                   |
| fgets(str,size,stdin) | 输入字符串     | 最多读 size-1 个字符，读空格，安全无溢出               |
| printf("%s",str)      | 输出字符串     | 遇'\\0'停止，需手动加\\n换行                           |
| puts(str)             | 输出字符串     | 遇'\\0'停止，****自动加**\\n**                         |
| fputs(str,stdout)     | 输出字符串     | 遇'\\0'停止，不自动换行，安全                          |
| strlen(str)           | 计算字符串长度 | 头文件string.h，****不包含**'\\0'**，返回 unsigned int |

### 5.5.4 常用操作（强化训练）

****字符串追加****：将一个字符串拼接到另一个字符串末尾，核心步骤：

. 遍历源字符串，复制到目标数组；
. 从源字符串结束位置继续遍历待追加字符串，拼接到目标数组；
. 在目标数组末尾手动添加'\\0'，保证为合法字符串。

# 第六章 函数

## 一、函数核心概述

1. **分类**：系统函数（库函数，如`printf()`）、用户定义函数（自定义功能）。

2. **核心作用**：减少代码重复、实现程序模块化（分工清晰，便于维护）。

3. **调用五要素**：头文件、函数名、功能、参数（类型 / 个数匹配）、返回值（按需接收）。

4. 典型案例（生成随机数）

   ```c
   #include <stdio.h>
   #include <time.h>
   #include <stdlib.h>
   int main() {
       srand((unsigned int)time(NULL)); // 设随机种子
       printf("随机数：%d\n", rand());
       retun 0;
   }
   ```

## 二、函数定义

### 1. 通用格式

```
返回类型 函数名(形式参数列表) {
    数据定义;
    执行语句;
}
```

### 2. 关键规则

- **函数名**：见名知意，后接`()`区分变量。
- **形参**：类型 + 变量形式（如`int a`），无参写`()`或`void`，不可直接赋值。
- **函数体**：`{}`内实现核心功能。
- 返回值
  - 有返回值：`return`后跟对应类型值，类型不一致时以函数返回类型为准。
  - 无返回值：函数前加`void`，`return`可省略（或仅`return;`）。
  - `return`可中断函数，后续代码不执行。

## 三、函数调用

### 1. 执行流程

main () 阻塞 → 查找函数定义 → 匹配参数 → 执行函数体 → 返回 main () 继续执行。

### 2. 调用规则

- **无参调用**：`函数名();`，不可加实参。
- **有参调用**：实参个数 / 类型与形参一致，可是常量、变量、表达式（需有确定值）。
- **值传递**：形参修改不影响实参（实参、形参是独立单元）。
- **返回值使用**：无返回值函数不可接收返回值；有返回值需用匹配类型变量接收。

## 四、函数声明

1. **场景**：被调函数在主调函数后，或跨文件调用。
2. **格式**：`返回类型 函数名(形参类型/名);`（分号必加），如`int max(int x, int y);`。
3. **区别**：定义含函数体（可执行），声明仅通知编译器（无执行代码）；一个函数可声明多次、定义一次。

## 五、main 与 exit 函数

- **main () 中**：`return 0;`与`exit(0);`均终止整个程序。
- **子函数中**：`return;`仅终止子函数，`exit(0);`终止整个程序（需含`stdlib.h`）。
- **注意**：使用`exit()`需引入头文件`stdlib.h`。

## 六、多文件编程

1. **规范**：.h 头文件存声明（函数 / 宏 / 结构体），.c 源文件存实现，main.c 为程序入口。

2. 防头文件重复包含

   - 方式 1（跨平台）：`#ifndef __XXX_H__ #define __XXX_H__ 声明内容 #endif`

     ```c
     #ifndef __XXX_H__ // 自定义宏名，一般为头文件名大写+下划线，保证唯一
     #define __XXX_H__
     
     // 头文件的声明内容（函数、宏、结构体等）
     
     #endif // 结束条件编译
     ```

   - 方式 2（简单）：`#pragma once 声明内容`

     ```c
     #pragma once // 直接写在头文件第一行，编译器保证该头文件仅被包含一次
     
     // 头文件的声明内容（函数、宏、结构体等）
     ```


# 第七章 指针笔记整理

## 7.1 指针核心概述

### 7.1.1 内存与地址基础

1. **内存定义**：计算机中暂存程序和数据的核心部件（掉电丢失），是 CPU 与外存（硬盘、U 盘等，掉电不丢）的桥梁，所有数据操作均需通过内存完成。

2. 内存地址

   ：内存被抽象为一维字符数组，每个字节分配唯一的 32 位（32 位平台）或 64 位（64 位平台）编号，该编号即为

   内存地址

   ，用于定位数据存储位置。

   - 不同类型数据占用字节数不同，地址分配规则：`char`（1 字节→1 个地址）、`int`（4 字节→4 个连续地址）等。

### 7.1.2 指针与指针变量的本质

- **指针**：本质是内存地址（如`0x7ffeefbff5ac`），是内存单元的唯一编号。
- **指针变量**：专门用于存放内存地址的变量，需声明对应数据类型（如`int *p`），类型决定指针操作内存的步长（如`int *`每次移动 4 字节）。
- 关键区别：指针是地址本身，指针变量是存放地址的容器，日常叙述中常将指针变量简称为 “指针”。

## 7.2 指针基础知识

### 7.2.1 指针变量的定义与使用

#### 1) 定义语法

```c
// 格式：数据类型 * 指针变量名;
int *p;    // 定义指向int类型的指针变量p
char *q;   // 定义指向char类型的指针变量q
```

- 说明：`*`表示 “指针类型”，变量名是`p`/`q`，而非`*p`/`*q`。

#### 2) 核心操作

- **取地址（&）**：获取变量的内存地址，赋值给指针变量（指针变量指向该变量）。
- **解引用（\*）**：通过指针变量存储的地址，访问或修改对应内存空间的值。

#### 3) 示例代码

```c
#include <stdio.h>
int main()
{
    int a = 10;
    char b = 'A';
    int *p = &a;    // p指向a的地址
    char *q = &b;   // q指向b的地址
    
    printf("a的地址：%p\n", &a);  // 输出a的地址，与p的值一致
    printf("*p = %d\n", *p);      // 解引用p，输出a的值：10
    printf("*q = %c\n", *q);      // 解引用q，输出b的值：A
    
    *p = 20;  // 通过指针修改a的值
    printf("修改后a = %d\n", a);  // 输出：20
    return 0;
}
```

- 注意：`&`不能用于寄存器变量（无内存地址）。

### 7.2.2 指针大小

- 指针大小与

  数据类型无关

  ，仅取决于操作系统位数：

  - 32 位平台：所有指针均为 4 字节（32 位地址）；
  - 64 位平台：所有指针均为 8 字节（64 位地址）。

- 示例验证：

```c
printf("sizeof(int *) = %zu\n", sizeof(int *));    // 4或8
printf("sizeof(char *) = %zu\n", sizeof(char *));  // 4或8
```

### 7.2.3 野指针与空指针

#### 1) 野指针（危险）

- 定义：未初始化、随意赋值或指向已释放内存的指针（如`int *p = 0x12345678`）。
- 风险：操作野指针指向的内存会导致程序崩溃（操作系统禁止访问未知内存）。
- 避免：指针定义后立即初始化（指向有效变量或设为 NULL）。

#### 2) 空指针（安全）

- 定义：赋值为`NULL`的指针，`NULL`是值为 0 的宏常量（`#define NULL ((void *)0)`），标志指针未指向任何有效内存。
- 用法：`int *p = NULL;`，避免野指针风险，使用前需判断是否为 NULL。

### 7.2.4 万能指针（void *）

- 特性：可指向任意类型变量的内存空间，但**不能直接解引用**，使用前需强制转换为对应类型。
- 示例：

```c
void *p = NULL;
int a = 10;
p = (void *)&a;          // 指向int变量a，强制转换为void *
*( (int *)p ) = 20;      // 强制转换为int *后解引用，修改a的值
printf("a = %d\n", a);   // 输出：20
```

### 7.2.5 const 修饰的指针变量

const 修饰指针有两种场景，核心区别在于 “限制指针指向” 还是 “限制指针指向的内存”：

| 定义形式                             | 含义（从右往左读） | 核心限制                                  |
| :----------------------------------- | :----------------- | :---------------------------------------- |
| `const int *p`（等价`int const *p`） | 指向常量的指针     | 不能通过`*p`修改内存值，`p`可指向其他地址 |
| `int *const p`                       | 指针常量           | `p`不能指向其他地址，可通过`*p`修改内存值 |
| `const int *const p`                 | 指向常量的指针常量 | 既不能改`p`的指向，也不能改`*p`的内存值   |

- 示例：

```c
int a = 10, b = 20;
const int *p1 = &a;
// *p1 = 30;  // 错误：不能修改指向的内存
p1 = &b;        // 正确：可指向其他地址

int *const p2 = &a;
// p2 = &b;  // 错误：不能改变指向
*p2 = 30;      // 正确：可修改内存值
```

## 7.3 指针与数组

### 7.3.1 数组名的本质

- 数组名是**数组首元素的地址常量**（不可修改，如`a = &b`错误），与`&a[0]`等价。
- 示例：

```c
int a[] = {1,2,3,4,5};
printf("a = %p\n", a);      // 输出首元素地址
printf("&a[0] = %p\n", &a[0]);  // 与a的值一致
```

### 7.3.2 指针操作数组元素

数组元素的访问有两种等价方式：`a[i]`与`*(a+i)`/`*(p+i)`（`p`是指向数组首元素的指针）。

- 示例：

```c
#include <stdio.h>
int main()
{
    int a[] = {1,2,3,4,5};
    int *p = a;  // p指向数组首元素，等价于p = &a[0]
    
    // 两种方式遍历数组，结果一致
    for (int i = 0; i < 5; i++)
    {
        printf("a[%d] = %d, *(p+%d) = %d\n", i, a[i], i, *(p+i));
    }
    return 0;
}
```

### 7.3.3 指针加减运算

指针加减运算的步长由**指针类型**决定（非简单整数加减）：

- `int *`：+1 → 移动 4 字节（1 个 int）；
- `char *`：+1 → 移动 1 字节（1 个 char）。

#### 1) 加法运算（遍历数组）

```c
int a[] = {1,2,3,4,5};
int *p = a;
while (p < a+5)  // 指针移动到数组末尾
{
    printf("%d ", *p);  // 输出：1 2 3 4 5
    p++;                // 指针向后移动1个int（4字节）
}
```

#### 2) 减法运算（反向遍历 / 计算元素间距）

```c
int a[] = {1,2,3,4,5};
int *p1 = &a[1], *p2 = &a[3];
printf("p2 - p1 = %d\n", p2 - p1);  // 输出：2（相差2个int元素）
printf("(int)p2 - (int)p1 = %d\n", (int)p2 - (int)p1);  // 输出：8（相差8字节）
```

### 7.3.4 指针数组

- 定义：数组的每个元素都是指针类型（本质是 “数组”，而非 “指针”）。
- 语法：`数据类型 * 数组名[数组长度];`
- 示例：

```c
#include <stdio.h>
int main()
{
    int a = 1, b = 2, c = 3;
    int *p[3] = {&a, &b, &c};  // 指针数组p，元素为int *类型
    
    for (int i = 0; i < 3; i++)
    {
        printf("%d ", *(p[i]));  // 输出：1 2 3
    }
    return 0;
}
```

## 7.4 多级指针

- 定义：指向指针变量的指针（用于存储指针变量的地址），常用二级指针，三级及以上极少使用。
- 语法：`数据类型 ** 指针变量名;`（二级指针）
- 示例：

```c
#include <stdio.h>
int main()
{
    int a = 10;
    int *p = &a;    // 一级指针：p指向a
    int **q = &p;   // 二级指针：q指向p
    
    printf("a = %d\n", a);        // 输出：10
    printf("*p = %d\n", *p);      // 输出：10（通过一级指针访问a）
    printf("**q = %d\n", **q);    // 输出：10（通过二级指针访问a）
    return 0;
}
```

- 核心逻辑：`*q` = `p`（指针变量 p 的值），`**q` = `*p` = `a`。

## 7.5 指针与函数

### 7.5.1 指针作为函数参数（地址传递）

- 作用：通过指针将变量地址传入函数，实现 “修改实参的值”（值传递无法修改实参）。
- 示例（交换两个变量的值）：

```c
#include <stdio.h>
// 地址传递：形参为指针，接收实参地址
void swap(int *x, int *y)
{
    int tmp = *x;
    *x = *y;
    *y = tmp;
}
int main()
{
    int a = 3, b = 5;
    swap(&a, &b);  // 传入a和b的地址
    printf("a = %d, b = %d\n", a, b);  // 输出：a=5, b=3（实参被修改）
    return 0;
}
```

- 对比值传递：`void swap(int x, int y)`仅修改形参，实参不变。

### 7.5.2 数组名作为函数参数

- 数组名作为参数时，会**退化为指针**（传递的是数组首元素地址，而非整个数组），函数内无法通过`sizeof`获取数组总长度（需手动传入）。
- 示例：

```c
#include <stdio.h>
// 形参int *a 等价于 int a[]
void printArray(int *a, int n)  // n为数组长度
{
    for (int i = 0; i < n; i++)
    {
        printf("%d ", a[i]);  // 等价于*(a+i)
    }
}
int main()
{
    int a[] = {1,2,3,4,5};
    int len = sizeof(a)/sizeof(a[0]);  // 计算数组长度
    printArray(a, len);  // 传入数组名（首元素地址）和长度
    return 0;
}
```

### 7.5.3 指针作为函数返回值

- 作用：函数返回内存地址（如堆区、静态变量区地址），需确保返回的地址有效（避免返回栈区地址，函数调用后栈区释放）。
- 示例（返回静态变量地址）：

```c
#include <stdio.h>
int *getStaticVal()
{
    static int a = 10;  // 静态变量，存储在数据区，生命周期与程序一致
    return &a;          // 返回a的地址
}
int main()
{
    int *p = getStaticVal();
    *p = 20;            // 修改静态变量的值
    printf("*p = %d\n", *p);  // 输出：20
    return 0;
}
```

## 7.6 指针与字符串

## 7.6.1 字符指针

- **核心逻辑**：字符串本质是`'\0'`结尾的字符数组，字符指针可直接指向字符串首地址（字符数组或字符串常量）。

- 关键特性：

  - 指向字符数组：可通过指针修改数组内容（如`*p = 'm'`）；
  - 指向字符串常量（如`char *p = "world"`）：常量存储在只读区，不可修改，仅能读取。

  

- **示例代码**：

```
char str[] = "hello world";
char *p = str;
*p = 'm'; // 合法，修改数组内容
p++;
printf("%s\n", p); // 输出：ello

char *q = "test"; // 指向只读字符串常量
// *q = 'T'; // 错误，不可修改
printf("%s\n", q); // 输出：test
```

## 7.6.2 字符指针做函数参数

- **核心用途**：传递字符串首地址，实现字符串拼接、拷贝、比较等功能，配合`const`可保护源字符串不被修改。
- **示例（字符串拼接）**：

```
void mystrcat(char *dest, const char *src) {
    int len1 = 0, len2 = 0;
    while (dest[len1]) len1++; // 移动到dest末尾
    while (src[len2]) dest[len1++] = src[len2++]; // 拼接src
    dest[len1] = '\0'; // 手动加结束符
}

int main() {
    char dst[100] = "hello mike";
    char src[] = "123456";
    mystrcat(dst, src);
    printf("dst = %s\n", dst); // 输出：hello mike123456
    return 0;
}
```

## 7.6.3 const 修饰的字符指针（常用场景）

| 定义形式              | 核心限制                                      | 适用场景                     |
| :-------------------- | :-------------------------------------------- | :--------------------------- |
| `const char *p`       | 不可通过`*p`修改字符串内容，`p`可指向其他地址 | 保护源字符串不被修改         |
| `char *const p`       | `p`不可指向其他地址，可修改`*p`内容           | 固定指向某一字符串，允许修改 |
| `const char *const p` | 既不可改指向，也不可改字符串内容              | 只读字符串，固定指向         |

- **示例代码**：

```
char buf[] = "aklgjdlsgjlkds";
const char *p1 = buf;
// p1[1] = '2'; // 错误：不可修改内容
p1 = "agdlsjaglkdsajgl"; // 正确：可修改指向

char *const p2 = buf;
p2[1] = '3'; // 正确：可修改内容
// p2 = "salkjgldsjaglk"; // 错误：不可修改指向
```

## 7.6.4 指针数组做为 main 函数的形参

- 标准格式：

  ```
  int main(int argc, char *argv[])
  ```

  - `argc`：命令行参数个数（包含程序名本身）；
  - `argv`：指针数组，每个元素指向一个命令行参数字符串。

- **示例代码**：

```
#include <stdio.h>
int main(int argc, char *argv[]) {
    printf("参数总数：%d\n", argc);
    for (int i = 0; i < argc; i++) {
        printf("参数%d：%s\n", i, argv[i]);
    }
    return 0;
}
```

- 运行说明：编译后执行

  ```
  ./a.out hello world
  ```

  ，输出：plaintext

  ```
  参数总数：3
  参数0：./a.out
  参数1：hello
  参数2：world
  ```

  

## 7.6.5 项目开发常用字符串应用模型

### 1) strstr 查找子串模型（统计出现次数）

**while 模型（推荐，逻辑简洁）**

```c
#include <stdio.h>
#include <string.h>
int count_substr(char *str, char *sub) {
    int cnt = 0;
    int len = strlen(sub);
    while ((str = strstr(str, sub)) != NULL) {
        cnt++;
        str += len; // 跳过当前子串，避免重复匹配
    }
    return cnt;
}

int main() {
    char *str = "11abcd111122abcd333abcd";
    printf("子串出现次数：%d\n", count_substr(str, "abcd")); // 输出：3
    return 0;
}
```

**do-while 模型（适配边界场景）**

```c
#include <stdio.h>
#include <string.h>
int count_substr_do(char *str, char *sub) {
    int cnt = 0;
    int len = strlen(sub);
    do {
        str = strstr(str, sub);
        if (str != NULL) {
            cnt++;
            str += len;
        } else {
            break;
        }
    } while (*str != 0);
    return cnt;
}
```

### 2) 两头堵模型（去除首尾空格，统计有效字符数）

```c
#include <stdio.h>
#include <string.h>
int trim_space(char *str, int *valid_len) {
    if (str == NULL || valid_len == NULL) return -1;
    int begin = 0, end = strlen(str) - 1;
    // 跳过左侧空格
    while (str[begin] == ' ' && str[begin]) begin++;
    // 跳过右侧空格
    while (str[end] == ' ' && end > 0) end--;
    // 无有效字符
    if (end < begin) {
        *valid_len = 0;
        return -2;
    }
    *valid_len = end - begin + 1;
    str[end + 1] = '\0'; // 截断空格
    return 0;
}

int main() {
    char str[] = "      abcddsgadsgefg      ";
    int valid_len = 0;
    trim_space(str, &valid_len);
    printf("有效字符串：%s\n", str); // 输出：abcddsgadsgefg
    printf("有效长度：%d\n", valid_len); // 输出：12
    return 0;
}
```

### 3) 字符串反转模型（双指针交换）

```c
#include <stdio.h>
#include <string.h>
void reverse_str(char *str) {
    if (str == NULL) return;
    int begin = 0, end = strlen(str) - 1;
    char tmp;
    while (begin < end) {
        // 交换字符
        tmp = str[begin];
        str[begin++] = str[end];
        str[end--] = tmp;
    }
}

int main() {
    char str[] = "abcdef"; // 字符数组，可修改
    reverse_str(str);
    printf("反转后：%s\n", str); // 输出：fedcba
    return 0;
}
```

# 第八章内存管理

## 8.1 作用域

C 语言变量 / 函数的作用域分为**代码块作用域**、**函数作用域**、**文件作用域**，核心区分各类变量的生效范围、生命周期，函数可通过`static`限制作用域。

### 8.1.1 局部变量（auto 自动变量）

- 定义：代码块`{}`内定义，`auto`关键字可省略，仅能在`{}`内使用
- 生命周期：函数 / 复合语句执行结束后销毁
- 初始化：未赋初值则内容为随机值
- 示例：if 语句内定义的变量，出 if 块后无法访问

### 8.1.2 静态 (static) 局部变量

- 作用域：仅在定义的函数 / 代码块内有效
- 生命周期：与程序运行周期一致，函数调用结束后不销毁
- 初始化：仅初始化一次；未赋初值时系统自动赋值（数值型 0、字符型空字符）
- 示例：多次调用函数，静态局部变量会保留上一次调用的结果

### 8.1.3 全局变量

- 定义：函数外定义，无关键字修饰
- 作用域：整个程序（本文件 + 其他文件），其他文件使用需用`extern`声明
- 生命周期：与程序运行周期一致
- 注意：不同文件的全局变量**不可重名**

### 8.1.4 静态 (static) 全局变量

- 定义：函数外定义，加`static`修饰
- 作用域：仅限制在定义的文件内
- 生命周期：与程序运行周期一致，仅初始化一次
- 注意：不同文件的静态全局变量**可重名**，作用域不冲突

### 8.1.5 extern 全局变量声明

- 语法：`extern 类型 变量名;`
- 作用：声明变量在**其他文件已定义**，仅做引用，不重新定义

### 8.1.6 全局函数和静态函数

- 全局函数：默认所有函数为全局，作用域整个程序，**不同文件不可重名**
- 静态函数：加`static`修饰，作用域仅定义的文件内，**不同文件可重名**，其他文件无法调用（即使声明也无效）

### 8.1.7 核心注意点

1. 不同函数可使用相同变量名，分配不同内存，互不干扰
2. 同一源文件中，全局变量和局部变量可同名，**局部作用域内全局变量失效**
3. 所有函数默认全局，static 函数为文件级作用域

### 8.1.8 作用域 & 生命周期总结表

| 类型            | 作用域     | 生命周期       |
| :-------------- | :--------- | :------------- |
| auto 变量       | 一对 {} 内 | 当前函数       |
| static 局部变量 | 一对 {} 内 | 整个程序运行期 |
| extern 变量     | 整个程序   | 整个程序运行期 |
| static 全局变量 | 当前文件   | 整个程序运行期 |
| extern 函数     | 整个程序   | 整个程序运行期 |
| static 函数     | 当前文件   | 整个程序运行期 |
| register 变量   | 一对 {} 内 | 当前函数       |
| 全局变量        | 整个程序   | 整个程序运行期 |

## 8.2 内存布局

C 代码经**预处理→编译→汇编→链接**生成可执行程序，程序**加载前**分 3 区，**加载后**新增栈区、堆区，共 5 大内存分区，各分区有固定存储内容和特性。

### 8.2.1 内存分区（加载前 + 加载后）

**程序加载前（可执行文件）**

仅包含**代码区 (text)**、**数据区 (data)**、**未初始化数据区 (bss)**，大小固定，运行期不可变。

1. 代码区：存放 CPU 执行的机器指令，**只读、可共享**，规划局部变量相关信息
2. 数据区（静态数据区）：存放**明确初始化的全局变量**、**已初始化的静态变量**（全局 + 局部）、**字符串常量**
3. bss 区：存放**全局未初始化变量**、**未初始化静态变量**，程序执行前由内核初始化为 0/NULL

**程序加载后（内存中）**

在加载前 3 区基础上，新增**栈区**、**堆区**，成为完整的 5 大内存分区。

1. 栈区 (stack)：先进后出，由编译器**自动分配 / 释放**，存放函数参数、返回值、局部变量；实时加载释放，局部变量生命周期与栈空间一致
2. 堆区 (heap)：无固定顺序，容量远大于栈，由程序员**手动分配 / 释放**（未释放则程序结束后由操作系统回收）；用于动态内存分配，位于 BSS 区和栈区之间

### 8.2.2 存储类型全总结表

（作用域 + 生命周期 + 存储位置）

| 类型            | 作用域     | 生命周期       | 存储位置                        |
| :-------------- | :--------- | :------------- | :------------------------------ |
| auto 变量       | 一对 {} 内 | 当前函数       | 栈区                            |
| static 局部变量 | 一对 {} 内 | 整个程序运行期 | 初始化→data 段，未初始化→BSS 段 |
| extern 变量     | 整个程序   | 整个程序运行期 | 初始化→data 段，未初始化→BSS 段 |
| static 全局变量 | 当前文件   | 整个程序运行期 | 初始化→data 段，未初始化→BSS 段 |
| extern 函数     | 整个程序   | 整个程序运行期 | 代码区                          |
| static 函数     | 当前文件   | 整个程序运行期 | 代码区                          |
| register 变量   | 一对 {} 内 | 当前函数       | 运行时存储在 CPU 寄存器         |
| 字符串常量      | 当前文件   | 整个程序运行期 | data 段                         |

### 8.2.3 内存操作函数

（头文件：`#include <string.h>`）

均为字节级操作，适用于所有数据类型，核心处理内存的填充、拷贝、比较。

1. **memset()**
   - 语法：`void *memset(void *s, int c, size_t n);`
   - 功能：将 s 首地址的前 n 个字节以字符 c 填充（c 为 unsigned char，范围 0~255）
   - 常用场景：动态内存初始化（如将数组清 0）
   - 返回值：s 的首地址
2. **memcpy()**
   - 语法：`void *memcpy(void *dest, const void *src, size_t n);`
   - 功能：将 src 首地址的前 n 个字节拷贝到 dest 首地址
   - 注意：**dest 和 src 内存空间不可重叠**，否则程序报错
   - 返回值：dest 的首地址
3. **memmove()**
   - 功能 / 用法：与 memcpy 完全一致
   - 区别：**支持 dest 和 src 内存空间重叠**，执行效率略低于 memcpy
4. **memcmp()**
   - 语法：`int memcmp(const void *s1, const void *s2, size_t n);`
   - 功能：比较 s1 和 s2 首地址的前 n 个字节
   - 返回值：相等→0；s1>s2→>0；s1<s2→<0

### 8.2.4 堆区内存分配和释放

（头文件：`#include <stdlib.h>`）

堆区内存需**手动分配、手动释放**，核心函数为 malloc () 和 free ()，配套 memset 做初始化。

1. **malloc()**
   - 语法：`void *malloc(size_t size);`
   - 功能：在堆区分配**size 字节的连续内存**，内容为随机值
   - 参数：size→需分配的内存大小（单位：字节）
   - 返回值：成功→分配空间的首地址（需强制类型转换）；失败→NULL
   - 注意：分配后建议用 memset 初始化，避免随机值问题
2. **free()**
   - 语法：`void free(void *ptr);`
   - 功能：释放 ptr 指向的堆区内存（ptr 为 malloc 分配的首地址）
   - 注意：① 不可多次释放同一内存；② 释放后建议将指针置 NULL，避免野指针；③ 仅能释放 malloc 分配的内存

## 8.3 内存分区代码分析

### 错误 1：返回栈区地址

函数内的局部变量存于栈区，函数调用结束后栈区内存释放，返回其地址会得到**野指针**，操作该内存会报错。

```c
int *fun() { int a = 10; return &a; } // a销毁，返回野指针
int main() { int *p = fun(); *p = 100; // 错误：操作野指针 }
```

### 正确 2：返回 data 区地址

静态局部变量存于 data/BSS 区，函数调用结束后不销毁，返回其地址可正常使用。

```c
int *fun() { static int a = 10; return &a; } // a不销毁
int main() { int *p = fun(); *p = 100; // 正确，*p=100 }
```

### 错误 3：值传递修改指针本身

函数形参为指针时，**值传递仅能修改指针指向的内容**，无法修改指针本身的地址，形参分配堆区内存对实参无影响，实参仍为 NULL。

```c
void fun(int *tmp) { tmp = malloc(sizeof(int)); *tmp = 100; }
int main() { int *p = NULL; fun(p); *p = 100; // 错误：操作空指针 }
```

### 正确 4：值传递修改指针指向的内容

实参先分配堆区内存，将指针地址传入函数，值传递可正常修改指针指向的堆区内容。

```c
void fun(int *tmp) { *tmp = 100; }
int main() { int *p = malloc(sizeof(int)); fun(p); // 正确，*p=100 }
```

### 正确 5：返回堆区地址

堆区内存由程序员管理，函数内分配堆区内存后返回其地址，函数调用结束后堆区内存不销毁，使用后需手动 free 释放。

```c
int *fun() { int *tmp = malloc(sizeof(int)); *tmp = 100; return tmp; }
int main() {
    int *p = fun(); printf("*p = %d\n", *p); // 正确，*p=100
    if (p != NULL) { free(p); p = NULL; } // 手动释放，置NULL避免野指针
```

# 第九章复合类型

## 9.1 结构体

### 9.1.1 概述

结构体是 C 语言**构造数据类型**，用于将**不同类型的数据**组合成一个有机整体（如学生的学号、姓名、年龄等属性），解决数组只能存放相同类型数据的问题，便于数据管理。

### 9.1.2 结构体变量的定义和初始化

结构体**类型**是数据模型，不分配内存；结构体**变量**基于类型创建，系统为其分配实际内存，有 3 种定义方式：

1. **先声明类型，再定义变量（常用）**

```c
// 声明结构体类型
struct stu {
    char name[50];
    int age;
};
// 定义变量并初始化
struct stu s1 = { "mike", 18 };
```

1. **声明类型同时定义变量**

```c
struct stu2 {
    char name[50];
    int age;
}s2 = { "lily", 22 }; // 直接定义变量s2
```

1. **直接定义变量（无类型名，仅能使用一次）**

```c
struct {
    char name[50];
    int age;
}s3 = { "yuri", 25 }; // 无类型名，无法复用
```

### 9.1.3 结构体成员的访问

- **普通结构体变量**：使用**点运算符 `.`** 访问成员
- **结构体指针变量**：使用**箭头运算符 `->`** 访问成员（等价于`(*指针).成员`）

```c
#include<stdio.h>
#include<string.h>
struct stu { char name[50]; int age; };
int main() {
    struct stu s1;
    // 普通变量访问
    strcpy(s1.name, "abc");
    s1.age = 18;
    // 指针访问（取地址后用->）
    strcpy((&s1)->name, "test");
    (&s1)->age = 22;
    return 0;
}
```

### 9.1.4 结构体数组

数组元素为结构体类型，用于存储一组同类型的结构体数据（如多个学生信息），通过**数组下标 +`.`** 访问成员，也可通过`数组名+偏移量->`访问。

```c
struct stu boy[5] = { {101, "Li ping", 'M', 45}, {102, "Zhang ping", 'M', 62.5} };
// 访问方式1：下标+点运算符
boy[i].score;
// 访问方式2：地址+箭头运算符
(boy+i)->score;
```

### 9.1.5 结构体嵌套

一个结构体的**成员**是另一个已定义的结构体类型，访问嵌套成员需**逐层使用`.`** （普通变量）或`->`（指针变量）。

```c
// 内层结构体
struct person { char name[20]; char sex; };
// 外层结构体嵌套person
struct stu { int id; struct person info; };
// 访问嵌套成员：逐层访问
struct stu s[2] = { 1, "lily", 'F', 2, "yuri", 'M' };
s[i].info.name; // 访问stu的info成员的name
```

### 9.1.6 结构体赋值

**相同类型的结构体变量**可直接相互赋值，系统会将源变量的**所有成员值逐字节拷贝**到目标变量，二者内存独立，修改其一不影响另一个（等价于`memcpy`拷贝）。

```c
struct stu s1 = { "abc", 18 };
struct stu s2 = s1; // 直接赋值，所有成员值拷贝
```

### 9.1.7 结构体和指针

#### 1) 指向普通结构体变量的指针

定义结构体指针，指向结构体变量的地址，通过`->`访问成员，`(*p).成员`与`p->成员`等价。

```c
struct stu s1 = { "lily", 18 };
struct stu *p = &s1;
p->name; // 等价于(*p).name
```

#### 2) 堆区结构体变量

通过`malloc`在堆区分配结构体内存，返回值强制转换为结构体指针，使用后需**手动 free 释放**，避免内存泄漏。

```c
struct stu *p = (struct stu *)malloc(sizeof(struct stu));
strcpy(p->name, "test"); // 堆区变量访问
free(p); p = NULL; // 释放并置空，避免野指针
```

#### 3) 结构体嵌套一级指针

若结构体成员包含指针，需**分两步分配内存**：先分配结构体本身的堆内存，再为指针成员分配堆内存；释放时**先释放指针成员，再释放结构体**，避免内存泄漏。

```c
struct stu { char *name; int age; }; // 成员为一级指针
struct stu *p = (struct stu *)malloc(sizeof(struct stu));
p->name = malloc(strlen("test") + 1); // 为指针成员分配内存
strcpy(p->name, "test");
// 释放顺序：先成员，后结构体
free(p->name); p->name = NULL;
free(p); p = NULL;
```

### 9.1.8 结构体做函数参数

结构体传参有**值传递**和**地址传递**两种方式，地址传递效率更高（仅传 4/8 字节地址，而非整个结构体数据），是实际开发的首选。

#### 1) 普通变量做参数（值传递）

函数接收结构体副本，**修改形参不会影响实参**，适合小型结构体，大型结构体易造成栈溢出。

```c
void set_stu(struct stu tmp) { strcpy(tmp.name, "mike"); } // 形参是副本
int main() { struct stu s = {0}; set_stu(s); } // 实参值不变
```

#### 2) 指针变量做参数（地址传递）

函数接收结构体地址，通过指针**直接操作实参的内存**，修改形参会同步影响实参，效率高。

```c
void set_stu_pro(struct stu *tmp) { strcpy(tmp->name, "mike"); }
int main() { struct stu s = {0}; set_stu_pro(&s); } // 实参被修改
```

#### 3) 结构体数组名做参数

数组名本质是首元素地址，传参时等价于结构体指针，函数内可通过指针遍历数组，修改数组元素的成员。

```c
void set_stu_pro(struct stu *tmp, int n) { // 数组名传参=指针
    for (int i=0; i<n; i++) { tmp->age = 20+i; tmp++; }
}
int main() { struct stu s[3] = {0}; set_stu_pro(s, 3); }
```

#### 4) const 修饰结构体指针形参

用于**保护实参不被修改**，根据`const`位置不同，保护效果不同，开发中常用于只读操作的函数形参：

- `struct stu * const p`：指针**指向不可变**，但指向的结构体成员**可修改**
- `const struct stu * p`：指针**指向可变**，但指向的结构体成员**不可修改**（常用）
- `const struct stu * const p`：指针**指向不可变**，成员也**不可修改**

## 9.2 共用体（联合体）

### 核心特性

1. 共用体是**同一块内存空间**存储不同类型数据的构造类型，所有成员**共享同一块内存**；
2. 共用体的**内存大小 = 最长成员的内存大小**（内存对齐）；
3. 同一时刻**只有一个成员有效**，新成员赋值会**覆盖原有成员的值**；
4. 共用体变量的地址和其**所有成员的地址完全相同**。

### 示例代码

```c
union Test {
    unsigned char a; // 1字节
    unsigned int b;  // 4字节
    unsigned short c;// 2字节
}; // 共用体大小为4字节（最长成员b）
int main() {
    union Test tmp;
    printf("%p, %p, %p\n", &tmp.a, &tmp.b, &tmp.c); // 地址完全相同
    tmp.b = 0x44332211; // 赋值int，覆盖所有成员
    printf("%x\n", tmp.a); // 11（低1字节）
    printf("%x\n", tmp.c); // 2211（低2字节）
    return 0;
}
```

**内存存储规则**：低字节存低地址，高字节存高地址。

## 9.3 枚举

### 核心特性

1. 枚举是将变量的**所有可能值一一列举**，变量值**仅限于列举的范围**，用于表示有限的状态（如星期、真假、颜色）；
2. 枚举值是**常量**，**不能在程序中赋值修改**；
3. 枚举元素的默认值是**从 0 开始的连续整数**，也可手动指定初始值，后续元素依次递增。

### 定义和使用

```c
// 手动指定sun=2，后续mon=3、tue=4...
enum weekday { sun = 2, mon, tue, wed, thu, fri, sat };
// 默认flase=0，true=1
enum bool { flase, true };
int main() {
    enum weekday a = sun; // 枚举变量只能赋值枚举元素
    enum bool flag = true;
    printf("%d, %d\n", a, flag); // 输出2, 1
    return 0;
}
```

## 9.4 typedef

### 核心作用

`typedef`是 C 语言关键字，用于**为已有数据类型定义一个新名字**（别名），**不能创建新类型**，仅简化类型书写，提高代码可读性和可移植性。

### 核心区别（与 #define）

| 特性     | typedef            | #define                |
| :------- | :----------------- | :--------------------- |
| 生效阶段 | 编译阶段           | 预处理阶段             |
| 作用范围 | 仅针对**数据类型** | 可定义常量、宏、别名等 |
| 语法要求 | 末尾必须加**分号** | 末尾不加分号           |
| 类型检查 | 编译器会做类型检查 | 简单字符串替换，无检查 |

### 使用场景

1.为基本类型定义别名

```c
typedef int INT;        // INT等价于int
typedef unsigned char UBYTE; // UBYTE等价于unsigned char
```

2.为自定义类型（结构体、枚举、共用体）定义别名，简化书写

```c
// 定义结构体并取别名TYPE，指针别名PTYPE
typedef struct type {
    UBYTE a;
    INT b;
}TYPE, *PTYPE;
// 直接用别名定义变量，无需写struct
TYPE t;
PTYPE p = &t; // 等价于struct type *p = &t;
```

# 第十章 文件操作

## 10.1 概述

### 10.1.1 磁盘文件和设备文件

C 语言中把**所有输入输出对象**都抽象为**文件**，分为两类：

- **磁盘文件**：存储在外部介质（磁盘、U 盘）的一组相关数据的有序集合，使用时调入内存，是实际的物理文件。
- **设备文件**：操作系统将主机相连的输入 / 输出设备（键盘、屏幕、打印机）抽象为文件，设备的输入 / 输出等价于对磁盘文件的读 / 写。

### 10.1.2 磁盘文件的分类

物理上，所有磁盘文件均以**字节为单位顺序存储**（二进制形式）；**逻辑上**按编码方式分为两类：

- **文本文件**：基于**字符编码**（ASCII、UNICODE），可通过文本编辑器直接打开，数据以字符形式存储。
- **二进制文件**：基于**值编码**，按数据在内存中的原始存储形式直接写入磁盘，不可直接用文本编辑器打开，存储更高效。

### 10.1.3 文本文件和二进制文件对比

以数字`5678`为例，两种文件的存储形式差异：

1. 文本文件

   ：拆分为单个字符存储，占 4 字节

   存储内容（ASCII 码）：

   ```
   00110101(5) 00110110(6) 00110111(7) 00111000(8)
   ```

2. 二进制文件

   ：按整数的二进制值存储，占 2 字节（int 型）

   存储内容（二进制码）：

   ```
   00010110 00101110
   ```

**核心区别**：文本文件易读、占空间大；二进制文件占空间小、读写效率高、适合存储非字符数据（如结构体、数组）。

## 10.2 文件的打开和关闭

文件操作遵循**先打开，后使用，最后关闭**的原则，核心依赖**文件指针**操作，打开后获取文件指针，关闭后释放指针资源。

### 10.2.1 文件指针

1. **定义**：C 语言用`FILE*`类型指针指向文件，`FILE`是系统通过`typedef`定义的结构体（包含文件名、文件状态、读写位置等信息），声明在`<stdio.h>`中。

2. **FILE 结构体核心成员**：缓冲区状态、文件描述符、缓冲区大小、当前读写指针等，用户无需手动修改，由系统维护。

3. 系统默认打开的 3 个特殊文件指针

   （无需定义，直接使用）：

   - `stdin`：标准输入，默认指向**键盘**（scanf、getchar 的默认输入源）。
   - `stdout`：标准输出，默认指向**屏幕**（printf、puts 的默认输出源）。
   - `stderr`：标准出错，默认指向**屏幕**（perror 的默认输出源）。

### 10.2.2 文件的打开（fopen）

#### 函数原型

```c
#include <stdio.h>
FILE * fopen(const char * filename, const char * mode);
```

#### 功能

打开指定文件，返回指向该文件的 FILE 指针，若打开失败返回`NULL`。

#### 参数

- ```
  filename
  ```

  ：文件名（可带

  相对路径 / 绝对路径

  ），路径分隔符建议用

  ```
  /
  ```

  （Windows/Linux 通用），

  ```
  \\
  ```

  仅适用于 Windows。

  - 相对路径：`"passwd.txt"`（当前目录）、`"./test/passwd.txt"`（当前目录子文件夹）、`"../passwd.txt"`（上级目录）。
  - 绝对路径：`"c:/test/passwd.txt"`（Windows）、`"/home/test/passwd.txt"`（Linux）。

  

- `mode`：文件打开模式，核心分为**文本模式**和**二进制模式**（`b`标识，仅 Windows 有效），常用模式如下：

| 打开模式 | 含义     | 关键特性                           |
| :------- | :------- | :--------------------------------- |
| r/rb     | 只读     | 不创建文件，文件不存在则报错       |
| w/wb     | 只写     | 文件存在则**清空**，不存在则创建   |
| a/ab     | 追加     | 从文件末尾写入，不存在则创建       |
| r+/rb+   | 读写     | 不创建新文件，文件不存在则报错     |
| w+/wb+   | 读写     | 文件存在则清空，不存在则创建       |
| a+/ab+   | 读写追加 | 读全文件，写仅在末尾，不存在则创建 |

#### 关键注意点（Windows/Linux 差异）

- `b`是二进制模式标识，**仅 Windows 有效**，Linux 下`r`和`rb`、`w`和`wb`效果一致。

- 换行符差异：Linux 文本文件行尾为

  ```
  \n
  ```

  ，Windows 为

  ```
  \r\n
  ```

  - Windows**文本模式**（不加`b`）：读文件时`\r\n`自动转`\n`，写文件时`\n`自动转`\r\n`。
  - Windows**二进制模式**（加`b`）：读写均不做换行符转换，按原始字节处理。
  - Linux：无文本 / 二进制模式差异，`\r\n`按两个普通字符处理。

  

#### 打开示例

```c
FILE *fp = NULL;
// 以只写模式打开上级目录的test文件，失败则打印错误信息
fp = fopen("../test", "w");
if (fp == NULL) {
    perror("open"); // 打印库函数出错原因（如文件不存在、权限不足）
    return -1;
}
```

### 10.2.3 文件的关闭（fclose）

#### 函数原型

```c
#include <stdio.h>
int fclose(FILE * stream);
```

#### 功能

关闭已打开的文件，将**缓冲区数据写入磁盘**，释放系统分配的文件资源（文件指针）。

#### 参数

- `stream`：需要关闭的文件指针（fopen 的返回值）。

#### 返回值

- 成功：返回`0`；失败：返回`-1`。

#### 关闭的必要性

1. 打开的文件会占用内存资源，不关闭会导致内存泄漏。
2. 一个进程同时打开的文件数有上限，超出后 fopen 会失败。
3. 若未手动关闭，程序退出时操作系统会统一关闭，但可能导致**缓冲区数据丢失**（未及时写入磁盘）。

#### 关闭示例

```c
fclose(fp); // 关闭文件指针
fp = NULL;  // 建议置空，避免野指针
```

## 10.3 文件的顺序读写

按**从文件开头到末尾**的顺序读写数据，核心分为 4 种方式，适配不同的读写场景，所有函数均声明在`<stdio.h>`中。

### 10.3.1 按字符读写：fputc（写）、fgetc（读）

#### 1. 写文件：fputc

```c
int fputc(int ch, FILE * stream);
```

- 功能：将字符`ch`（转换为 unsigned char）写入`stream`指向的文件。

- 返回值：成功返回写入的字符；失败返回`-1`。

- 示例：将字符串逐个字符写入文件

  ```c
  char buf[] = "this is a test for fputc";
  for (int i = 0; i < strlen(buf); i++) {
      fputc(buf[i], fp);
  }
  ```

#### 2. 文件结束判断：EOF、feof

- **EOF**：文件结束符，定义为`#define EOF (-1)`，仅适用于**文本文件**（ASCII 码无 - 1，可作为结束标志）。

- feof：适配文本 / 二进制文件的结束判断函数，判断最后一次读操作是否到达文件末尾。

  ```c
  int feof(FILE * stream);
  ```

  - 返回值：非 0（到达末尾）；0（未到达末尾）。

#### 3. 读文件：fgetc

```c
int fgetc(FILE * stream);
```

- 功能：从`stream`指向的文件中读取**一个字符**。

- 返回值：成功返回读取的字符（ASCII 码）；失败 / 到达末尾返回`-1`。

- 示例：读取文件所有字符并打印

  ```c
  char ch;
  while (!feof(fp)) { // 未到文件末尾则循环
      ch = fgetc(fp);
      printf("%c", ch);
  }
  ```

  

### 10.3.2 按行读写：fputs（写）、fgets（读）

适合读写**文本文件**的行数据（如配置文件、日志文件）。

#### 1. 写文件：fputs

```c
int fputs(const char * str, FILE * stream);
```

- 功能：将字符串`str`写入文件，**字符串结束符`\0`不写入**。

- 返回值：成功返回`0`；失败返回`-1`。

- 示例：逐行写入字符串

  ```c
  char *buf[] = { "123456\n", "bbbbbbbbbb\n" };
  for (int i = 0; i < 2; i++) {
      fputs(buf[i], fp);
  }
  ```

#### 2. 读文件：fgets

```c
char * fgets(char * str, int size, FILE * stream);
```

- 功能：从文件中读取字符到`str`，直到遇到**换行符`\n`、文件末尾、读取了`size-1`个字符**，自动在末尾加`\0`。

- 参数：`size`为最大读取长度（实际读`size-1`个字符，预留`\0`位置）。

- 返回值：成功返回`str`；到达末尾 / 失败返回`NULL`。

- 示例：逐行读取文件内容

  ```c
  char buf[100] = {0};
  while (!feof(fp)) {
      memset(buf, 0, sizeof(buf)); // 清空缓冲区
      char *p = fgets(buf, sizeof(buf), fp);
      if (p != NULL) {
          printf("buf = %s", buf);
      }
  }
  ```

### 10.3.3 格式化读写：fprintf（写）、fscanf（读）

按**指定格式**读写数据，与`printf/scanf`用法一致，仅多了文件指针参数，适合读写结构化的文本数据。

#### 1. 写文件：fprintf

```c
int fprintf(FILE * stream, const char * format, ...);
```

- 功能：按`format`格式将数据格式化后写入文件。

- 返回值：成功返回实际写入的字符数；失败返回`-1`。

- 示例：写入整数、字符串等格式化数据

  ```
  fprintf(fp, "name:%s, age:%d, score:%.1f\n", "mike", 18, 95.5);
  ```

#### 2. 读文件：fscanf

```c
int fscanf(FILE * stream, const char * format, ...);
```

- 功能：从文件中按`format`格式读取数据，存入对应变量（需传地址）。

- 返回值：成功返回**成功转换的参数个数**；失败 / 到达末尾返回`-1`。

- 示例：读取格式化数据到变量

  ```c
  char name[20];
  int age;
  float score;
  fscanf(fp, "name:%s, age:%d, score:%.1f\n", name, &age, &score);
  ```

### 10.3.4 按块读写：fwrite（写）、fread（读）

**以数据块为单位**读写，是**二进制文件**的核心读写方式，也可用于文本文件，读写效率最高，适合存储结构体、数组等连续数据。

#### 1. 写文件：fwrite

```c
size_t fwrite(const void *ptr, size_t size, size_t nmemb, FILE *stream);
```

- 功能：将`ptr`指向的内存数据，按**块大小`size`、块数`nmemb`** 写入文件。

- 参数：

  - `ptr`：待写入数据的内存首地址；
  - `size`：单个数据块的字节大小；
  - `nmemb`：要写入的块数；
  - 总写入字节数 = `size * nmemb`。

  

- 返回值：成功返回**实际写入的块数**（等于 nmemb）；失败返回`0`。

- 示例：将结构体数组写入文件

  ```c
  typedef struct Stu { char name[50]; int id; } Stu;
  Stu s[3] = {{"stu0",1}, {"stu1",2}, {"stu2",3}};
  fwrite(s, sizeof(Stu), 3, fp); // 每次写1个结构体，共写3个
  ```

#### 2. 读文件：fread

```c
size_t fread(void *ptr, size_t size, size_t nmemb, FILE *stream);
```

- 功能：从文件中按**块大小`size`、块数`nmemb`** 读取数据，存入`ptr`指向的内存。

- 返回值：成功返回**实际读取的块数**；若小于 nmemb 且大于 0，说明读到文件末尾；失败返回`0`。

- 示例：从文件读取结构体数组

  ```c
  Stu s[3];
  size_t ret = fread(s, sizeof(Stu), 3, fp);
  for (int i = 0; i < ret; i++) {
      printf("name:%s, id:%d\n", s[i].name, s[i].id);
  }
  ```

## 10.4 文件的随机读写

不按顺序，通过**移动文件读写光标**，直接定位到文件任意位置进行读写，核心依赖 3 个函数，适用于需要快速访问文件指定位置数据的场景。

### 10.4.1 移动读写光标：fseek

```c
int fseek(FILE *stream, long offset, int whence);
```

#### 功能

将文件的**读写光标**从`whence`位置移动`offset`个字节。

#### 参数

- `offset`：偏移量，正数**向后移**，负数**向前移**；超出文件开头报错，超出文件末尾则写入时扩大文件。

- ```
  whence
  ```

  ：偏移起始位置（系统定义的宏）：

  - `SEEK_SET`：文件**开头**（常用）；
  - `SEEK_CUR`：光标**当前位置**；
  - `SEEK_END`：文件**末尾**。

  

#### 返回值

成功返回`0`；失败返回`-1`。

#### 示例

```c
// 从文件开头，向后移动2个Stu结构体的位置（定位到第3个结构体）
fseek(fp, 2 * sizeof(Stu), SEEK_SET);
```

### 10.4.2 获取光标位置：ftell

```c
long ftell(FILE *stream);
```

#### 功能

获取当前文件读写光标的**偏移量**（从文件开头到当前位置的字节数）。

#### 返回值

成功返回偏移量（long 型）；失败返回`-1`。

#### 用途

结合 fseek 可**计算文件大小**：

```c
fseek(fp, 0, SEEK_END); // 光标移到文件末尾
long file_size = ftell(fp); // 获取末尾偏移量，即文件大小
```

### 10.4.3 光标移回开头：rewind

c

```c
void rewind(FILE *stream);
```

#### 功能

将文件读写光标**直接移到文件开头**，等价于`fseek(fp, 0, SEEK_SET)`，无返回值。

#### 示例

```c
rewind(fp); // 光标回到开头，可重新读取文件
```

### 10.4.4 随机读写综合示例

```c
typedef struct Stu { char name[50]; int id; } Stu;
Stu tmp;
// 定位到第3个结构体
fseek(fp, 2 * sizeof(Stu), SEEK_SET);
// 读取第3个结构体
fread(&tmp, sizeof(Stu), 1, fp);
printf("name:%s, id:%d\n", tmp.name, tmp.id);
// 光标移回开头，重新读取所有数据
rewind(fp);
```

## 10.5 Windows 和 Linux 文本文件核心区别

1. **换行符**：Linux 为`\n`（1 字节），Windows 为`\r\n`（2 字节）。
2. **二进制模式标识`b`**：仅 Windows 有效，Linux 下忽略`b`，文本 / 二进制模式无差异。
3. **换行符转换**：Windows 文本模式（不加`b`）自动做`\r\n`和`\n`的互转，二进制模式不转换；Linux 无转换。
4. **文件格式判断方法**：以**二进制模式**打开文件，读取行尾字符，若`\n`前是`\r`则为 Windows 格式，否则为 Linux 格式（示例代码见原文档）。

## 10.6 获取文件状态（stat）

获取文件的**属性信息**（大小、权限、修改时间、类型等），需包含头文件`<sys/types.h>`和`<sys/stat.h>`，适用于 Linux/Windows（MinGW 环境）。

### 函数原型

```c
int stat(const char *path, struct stat *buf);
```

### 功能

将`path`指定文件的状态信息，存入`buf`指向的`struct stat`结构体。

### 返回值

成功返回`0`；失败返回`-1`。

### 核心结构体（struct stat）成员

| 成员     | 含义                       |
| :------- | :------------------------- |
| st_size  | 文件大小（字节）           |
| st_mtime | 最后一次修改时间（时间戳） |
| st_mode  | 文件类型和存取权限         |
| st_uid   | 文件所属用户 ID            |
| st_gid   | 文件所属组 ID              |

### 示例：获取文件大小

```c
#include <sys/types.h>
#include <sys/stat.h>
#include <stdio.h>
int main(int argc, char **args) {
    struct stat st = {0};
    stat(args[1], &st); // args[1]为传入的文件名
    printf("文件大小：%ld 字节\n", st.st_size);
    return 0;
}
```

## 10.8 文件缓冲区

### 10.8.1 缓冲文件系统

ANSI C 标准采用**缓冲文件系统**处理文件，系统会为每个打开的文件在**内存中开辟一块缓冲区**，程序与磁盘的交互均通过缓冲区完成，而非直接读写磁盘。

- **写文件**：程序数据先写入**内存缓冲区**，缓冲区装满 / 文件关闭 / 手动刷新时，一次性写入磁盘。

- 读文件：磁盘数据先一次性读入内存缓冲区，程序从缓冲区逐个读取数据。

  核心目的：减少磁盘 I/O 操作次数，提高文件读写效率（磁盘读写远慢于内存）。

### 10.8.2 手动刷新缓冲区：fflush

默认情况下，缓冲区只有装满或文件关闭时才会写入磁盘，若需要**立即将缓冲区数据写入磁盘**，使用 fflush 函数。

#### 函数原型

```c
#include <stdio.h>
int fflush(FILE *stream);
```

#### 功能

刷新文件缓冲区，将缓冲区中未写入磁盘的数据**立即写入**。

#### 参数

- `stream`：文件指针；若为`NULL`，刷新所有打开文件的缓冲区。

#### 返回值

成功返回`0`；失败返回`-1`。

#### 示例

```c
fprintf(fp, "hello world");
fflush(fp); // 立即将缓冲区数据写入磁盘，无需等待缓冲区装满
```

**常用场景**：写日志文件、实时数据存储，避免程序异常退出导致缓冲区数据丢失。
