# C 语言内存管理 核心笔记

## 8.1 作用域

C 语言变量 / 函数的作用域分为**代码块作用域**、**函数作用域**、**文件作用域**，核心区分各类变量的生效范围、生命周期，函数可通过`static`限制作用域。

### 8.1.1 局部变量（auto 自动变量）

- 定义：代码块`{}`内定义，`auto`关键字可省略，仅能在`{}`内使用
- 生命周期：函数 / 复合语句执行结束后销毁
- 初始化：未赋初值则内容为随机值
- 示例：if 语句内定义的变量，出 if 块后无法访问

### 8.1.2 静态 (static) 局部变量

- 作用域：仅在定义的函数 / 代码块内有效
- 生命周期：与程序运行周期一致，函数调用结束后不销毁
- 初始化：仅初始化一次；未赋初值时系统自动赋值（数值型 0、字符型空字符）
- 示例：多次调用函数，静态局部变量会保留上一次调用的结果

### 8.1.3 全局变量

- 定义：函数外定义，无关键字修饰
- 作用域：整个程序（本文件 + 其他文件），其他文件使用需用`extern`声明
- 生命周期：与程序运行周期一致
- 注意：不同文件的全局变量**不可重名**

### 8.1.4 静态 (static) 全局变量

- 定义：函数外定义，加`static`修饰
- 作用域：仅限制在定义的文件内
- 生命周期：与程序运行周期一致，仅初始化一次
- 注意：不同文件的静态全局变量**可重名**，作用域不冲突

### 8.1.5 extern 全局变量声明

- 语法：`extern 类型 变量名;`
- 作用：声明变量在**其他文件已定义**，仅做引用，不重新定义

### 8.1.6 全局函数和静态函数

- 全局函数：默认所有函数为全局，作用域整个程序，**不同文件不可重名**
- 静态函数：加`static`修饰，作用域仅定义的文件内，**不同文件可重名**，其他文件无法调用（即使声明也无效）

### 8.1.7 核心注意点

1. 不同函数可使用相同变量名，分配不同内存，互不干扰
2. 同一源文件中，全局变量和局部变量可同名，**局部作用域内全局变量失效**
3. 所有函数默认全局，static 函数为文件级作用域

### 8.1.8 作用域 & 生命周期总结表

| 类型            | 作用域     | 生命周期       |
| :-------------- | :--------- | :------------- |
| auto 变量       | 一对 {} 内 | 当前函数       |
| static 局部变量 | 一对 {} 内 | 整个程序运行期 |
| extern 变量     | 整个程序   | 整个程序运行期 |
| static 全局变量 | 当前文件   | 整个程序运行期 |
| extern 函数     | 整个程序   | 整个程序运行期 |
| static 函数     | 当前文件   | 整个程序运行期 |
| register 变量   | 一对 {} 内 | 当前函数       |
| 全局变量        | 整个程序   | 整个程序运行期 |

## 8.2 内存布局

C 代码经**预处理→编译→汇编→链接**生成可执行程序，程序**加载前**分 3 区，**加载后**新增栈区、堆区，共 5 大内存分区，各分区有固定存储内容和特性。

### 8.2.1 内存分区（加载前 + 加载后）

**程序加载前（可执行文件）**

仅包含**代码区 (text)**、**数据区 (data)**、**未初始化数据区 (bss)**，大小固定，运行期不可变。

1. 代码区：存放 CPU 执行的机器指令，**只读、可共享**，规划局部变量相关信息
2. 数据区（静态数据区）：存放**明确初始化的全局变量**、**已初始化的静态变量**（全局 + 局部）、**字符串常量**
3. bss 区：存放**全局未初始化变量**、**未初始化静态变量**，程序执行前由内核初始化为 0/NULL

**程序加载后（内存中）**

在加载前 3 区基础上，新增**栈区**、**堆区**，成为完整的 5 大内存分区。

1. 栈区 (stack)：先进后出，由编译器**自动分配 / 释放**，存放函数参数、返回值、局部变量；实时加载释放，局部变量生命周期与栈空间一致
2. 堆区 (heap)：无固定顺序，容量远大于栈，由程序员**手动分配 / 释放**（未释放则程序结束后由操作系统回收）；用于动态内存分配，位于 BSS 区和栈区之间

### 8.2.2 存储类型全总结表

（作用域 + 生命周期 + 存储位置）

| 类型            | 作用域     | 生命周期       | 存储位置                        |
| :-------------- | :--------- | :------------- | :------------------------------ |
| auto 变量       | 一对 {} 内 | 当前函数       | 栈区                            |
| static 局部变量 | 一对 {} 内 | 整个程序运行期 | 初始化→data 段，未初始化→BSS 段 |
| extern 变量     | 整个程序   | 整个程序运行期 | 初始化→data 段，未初始化→BSS 段 |
| static 全局变量 | 当前文件   | 整个程序运行期 | 初始化→data 段，未初始化→BSS 段 |
| extern 函数     | 整个程序   | 整个程序运行期 | 代码区                          |
| static 函数     | 当前文件   | 整个程序运行期 | 代码区                          |
| register 变量   | 一对 {} 内 | 当前函数       | 运行时存储在 CPU 寄存器         |
| 字符串常量      | 当前文件   | 整个程序运行期 | data 段                         |

### 8.2.3 内存操作函数

（头文件：`#include <string.h>`）

均为字节级操作，适用于所有数据类型，核心处理内存的填充、拷贝、比较。

1. **memset()**
   - 语法：`void *memset(void *s, int c, size_t n);`
   - 功能：将 s 首地址的前 n 个字节以字符 c 填充（c 为 unsigned char，范围 0~255）
   - 常用场景：动态内存初始化（如将数组清 0）
   - 返回值：s 的首地址
2. **memcpy()**
   - 语法：`void *memcpy(void *dest, const void *src, size_t n);`
   - 功能：将 src 首地址的前 n 个字节拷贝到 dest 首地址
   - 注意：**dest 和 src 内存空间不可重叠**，否则程序报错
   - 返回值：dest 的首地址
3. **memmove()**
   - 功能 / 用法：与 memcpy 完全一致
   - 区别：**支持 dest 和 src 内存空间重叠**，执行效率略低于 memcpy
4. **memcmp()**
   - 语法：`int memcmp(const void *s1, const void *s2, size_t n);`
   - 功能：比较 s1 和 s2 首地址的前 n 个字节
   - 返回值：相等→0；s1>s2→>0；s1<s2→<0

### 8.2.4 堆区内存分配和释放

（头文件：`#include <stdlib.h>`）

堆区内存需**手动分配、手动释放**，核心函数为 malloc () 和 free ()，配套 memset 做初始化。

1. **malloc()**
   - 语法：`void *malloc(size_t size);`
   - 功能：在堆区分配**size 字节的连续内存**，内容为随机值
   - 参数：size→需分配的内存大小（单位：字节）
   - 返回值：成功→分配空间的首地址（需强制类型转换）；失败→NULL
   - 注意：分配后建议用 memset 初始化，避免随机值问题
2. **free()**
   - 语法：`void free(void *ptr);`
   - 功能：释放 ptr 指向的堆区内存（ptr 为 malloc 分配的首地址）
   - 注意：① 不可多次释放同一内存；② 释放后建议将指针置 NULL，避免野指针；③ 仅能释放 malloc 分配的内存

## 8.3 内存分区代码分析

### 错误 1：返回栈区地址

函数内的局部变量存于栈区，函数调用结束后栈区内存释放，返回其地址会得到**野指针**，操作该内存会报错。

```c
int *fun() { int a = 10; return &a; } // a销毁，返回野指针
int main() { int *p = fun(); *p = 100; // 错误：操作野指针 }
```

### 正确 2：返回 data 区地址

静态局部变量存于 data/BSS 区，函数调用结束后不销毁，返回其地址可正常使用。

```c
int *fun() { static int a = 10; return &a; } // a不销毁
int main() { int *p = fun(); *p = 100; // 正确，*p=100 }
```

### 错误 3：值传递修改指针本身

函数形参为指针时，**值传递仅能修改指针指向的内容**，无法修改指针本身的地址，形参分配堆区内存对实参无影响，实参仍为 NULL。

```c
void fun(int *tmp) { tmp = malloc(sizeof(int)); *tmp = 100; }
int main() { int *p = NULL; fun(p); *p = 100; // 错误：操作空指针 }
```

### 正确 4：值传递修改指针指向的内容

实参先分配堆区内存，将指针地址传入函数，值传递可正常修改指针指向的堆区内容。

```c
void fun(int *tmp) { *tmp = 100; }
int main() { int *p = malloc(sizeof(int)); fun(p); // 正确，*p=100 }
```

### 正确 5：返回堆区地址

堆区内存由程序员管理，函数内分配堆区内存后返回其地址，函数调用结束后堆区内存不销毁，使用后需手动 free 释放。

```c
int *fun() { int *tmp = malloc(sizeof(int)); *tmp = 100; return tmp; }
int main() {
    int *p = fun(); printf("*p = %d\n", *p); // 正确，*p=100
    if (p != NULL) { free(p); p = NULL; } // 手动释放，置NULL避免野指针
```
